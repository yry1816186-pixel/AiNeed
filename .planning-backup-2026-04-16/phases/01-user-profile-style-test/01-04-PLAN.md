**Extend existing AuthStore** in `stores/index.ts`:
- Add `wechatLogin(): Promise<void>`:
  - Call wechatAuth.authorize() to get code
  - Call auth apiClient.post('/auth/wechat', { code })
  - On success, set tokens and user in store (same pattern as existing login)
- Add `phoneLogin(phone: string, code: string): Promise<void>`:
  - Call smsApi.loginWithPhone(phone, code)
  - On success, set tokens and user
- Add `phoneRegister(phone: string, code: string, nickname?: string): Promise<void>`:
  - Call smsApi.registerWithPhone(phone, code, nickname)
  - On success, set tokens and user

**Create `stores/styleQuizStore.ts`**:
```typescript
interface StyleQuizStore {
  currentQuiz: QuizData | null;
  progress: { questionIndex: number; answers: Record<string, string> };
  result: QuizResult | null;
  isLoading: boolean;
  error: string | null;

  loadQuiz: (quizId: string) => Promise<void>;
  selectAnswer: (questionId: string, optionId: string) => Promise<void>;
  submitAll: (quizId: string) => Promise<void>;
  loadProgress: (quizId: string) => Promise<void>;
  reset: () => void;
}
```
- loadQuiz: call styleQuizApi.getQuiz, set currentQuiz
- selectAnswer: add to progress.answers, call styleQuizApi.saveProgress
- submitAll: call styleQuizApi.batchSubmit with all answers, set result
- loadProgress: call styleQuizApi.getProgress, restore questionIndex and answers
- Persist progress to AsyncStorage (use zustand/middleware persist)
- Reset clears all state

**Create `stores/profileStore.ts`**:
```typescript
interface ProfileStore {
  profile: UserProfile | null;
  completeness: { percentage: number; missingFields: string[] } | null;
  bodyAnalysis: BodyAnalysis | null;
  colorAnalysis: ColorAnalysis | null;
  isLoading: boolean;
  error: string | null;

  loadProfile: () => Promise<void>;
  loadCompleteness: () => Promise<void>;
  loadBodyAnalysis: () => Promise<void>;
  loadColorAnalysis: () => Promise<void>;
  updateProfile: (data: Partial<UserProfile>) => Promise<void>;
  refreshAll: () => Promise<void>;
}
```
- Each load method calls the corresponding profileApi method
- refreshAll: loads profile + completeness + analyses in parallel (Promise.all)
- Use TanStack Query for server state where appropriate (stores for client-side UI state)
