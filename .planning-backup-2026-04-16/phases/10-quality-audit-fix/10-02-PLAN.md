The audit found 29 hardcoded hex values in ClothingDetailScreen.tsx. Migrate all to theme tokens.

Common mappings to apply:

Background colors:
- '#FFFFFF' -> theme.colors.surface or DesignTokens.colors.neutral[50]
- '#F5F5F5', '#FAFAF8' -> theme.colors.background or DesignTokens.colors.neutral[100]
- '#000000' with opacity -> DesignTokens.colors.neutral[900] with opacity

Text colors:
- '#1A1A1A', '#333333', '#000000' -> theme.colors.text
- '#666666', '#888888' -> theme.colors.textSecondary
- '#999999', '#AAAAAA' -> theme.colors.textTertiary

Brand/accent colors:
- Any warm orange/red -> theme.colors.primary (Terracotta #C67B5C)
- Success green -> DesignTokens.colors.semantic.success
- Error red -> DesignTokens.colors.semantic.error

Border colors:
- '#E0E0E0', '#EEEEEE' -> theme.colors.border or DesignTokens.colors.neutral[200]
- '#DDDDDD' -> theme.colors.divider

Shadow colors:
- Keep '#000' for shadowColor (platform standard)

Steps:
1. Ensure `import { theme } from '../theme'` or `import { DesignTokens } from '../theme/tokens/design-tokens'` is at top
2. Find all hex color literals in StyleSheet.create and inline styles
3. Replace each with the appropriate token
4. Test TypeScript compilation
