## Task 2: Remove Duplicates and Establish Re-exports

<read_first>
- apps/mobile/src/components/ui/index.tsx
- apps/mobile/src/components/primitives/index.ts
- apps/mobile/src/components/primitives/Button/Button.tsx
- apps/mobile/src/components/primitives/Card/Card.tsx
- The audit results from Task 1
</read_first>

<action>
1. For each **Duplicate** component identified in Task 1:
   - Remove the inline implementation from `ui/index.tsx`
   - Replace with a re-export from primitives:
     ```typescript
     // Before: inline implementation of Button
     // After:
     export { Button } from '../primitives/Button/Button';
     export type { ButtonProps } from '../primitives/Button/Button';
     ```

2. For each **Composition** component:
   - Keep in `ui/index.tsx` but refactor to explicitly wrap the primitive:
     ```typescript
     // Example: GradientButton wraps primitive Button with gradient
     export const GradientButton: React.FC<GradientButtonProps> = (props) => {
       return <Button {...props} style={[props.style, gradientStyles]} />;
     };
     ```
   - Ensure it imports from primitives, not re-implements

3. For each **Unique** component that doesn't exist in primitives:
   - Move to `apps/mobile/src/components/primitives/` with proper file structure
   - Create index.ts barrel export
   - Re-export from `ui/index.tsx`

4. Verify the re-export list in `ui/index.tsx` includes:
   - All primitives (Button, Card, Dialog, EmptyState, Input, LoadingStates, Toast)
   - All composition components (GradientButton, OutfitCard, ProductGrid, BottomSheets, ChatBubble, etc.)
   - All transition and interaction components (PageTransitions, MicroInteractions, etc.)
   - All theme components (PaperThemeProvider, ThemeSystem, etc.)

5. Add a clear comment section at the top of `ui/index.tsx`:
   ```typescript
   /**
    * UI Component Index
    * 
    * Layering:
    * - primitives/ → Design system atoms (Button, Card, Input, etc.)
    * - ui/ → Composition components (GradientButton, OutfitCard, etc.)
    * - features/ → Business components (SwipeCard, WaterfallFeed, etc.)
    * 
    * This file RE-EXPORTS from primitives and provides composition components.
    * It does NOT re-implement primitives.
    */
   ```
</action>

- `grep -c "const Button" apps/mobile/src/components/ui/index.tsx` returns 0 (no inline Button definition)
- `grep -c "const Card" apps/mobile/src/components/ui/index.tsx` returns 0 (no inline Card definition)
