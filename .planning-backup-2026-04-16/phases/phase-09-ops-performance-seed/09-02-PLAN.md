# Phase 9 Plan 02: Content Review System + Enhanced Moderation

## Objective
Build the AI + human dual-track content review system. AI auto-reviews content on submission, flagged or uncertain content goes to human review queue. Moderators can approve/reject/delete content with full audit trail.

## Context
- ContentModerationService already exists in community module
- AdminCommunityController has basic moderation endpoints (moderatePost, getModerationQueue)
- AI Safety module exists at `apps/backend/src/modules/ai-safety/`
- Need to build the full review pipeline: AI pre-screen -> human queue -> approve/reject

## Tasks

### Task 1: Create Content Review Service

<task type="auto">
<read_first>
  - apps/backend/src/modules/community/content-moderation.service.ts
  - apps/backend/src/modules/ai-safety/ (list files)
</read_first>
<action>
Create `apps/backend/src/modules/admin/services/content-review.service.ts`:

1. Injectable service with PrismaService, AdminAuditService dependencies
2. `getReviewQueue(filters)` method:
   - Paginated list of content with moderationStatus = 'pending' or 'flagged'
   - Support filter by contentType (post/comment), priority (high/medium/low), dateRange
   - Include author info and AI moderation result if available
   - Sort by priority (flagged first), then by createdAt ascending (oldest first)

3. `reviewContent(id, reviewerId, action, note)` method:
   - action: 'approve' | 'reject' | 'delete'
   - Updates moderationStatus accordingly
   - Records reviewer and timestamp
   - Logs to AdminAuditService
   - If rejected/deleted: optionally notify author

4. `batchReview(ids, reviewerId, action, note)` method:
   - Bulk approve/reject for efficiency
   - Same logic as reviewContent but in a loop with transaction

5. `getReviewStats()` method:
   - Total pending, total reviewed today/this week/this month
   - Average review time (from created to reviewed)
   - Approval rate, rejection rate
   - Reviewer productivity (reviews per reviewer)
</action>
<acceptance_criteria>
  - ContentReviewService with all methods
  - Efficient queue queries with proper pagination
  - Audit logging on all review actions
</acceptance_criteria>
</task>
