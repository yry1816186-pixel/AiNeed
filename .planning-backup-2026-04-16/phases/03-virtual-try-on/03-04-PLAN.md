# Plan 04: 对象存储集成与 ML 服务换装路由

## Objective
完善 MinIO 对象存储集成（CDN 加速、水印图生成），更新 ML 服务虚拟试衣路由（替换 IDM-VTON 为 Doubao-Seedream + GLM），实现结果自动归档到灵感衣橱。

## Tasks

### Task 1: MinIO 存储增强 — 水印图与 CDN

<read_first>
- apps/backend/src/common/storage/storage.service.ts
</read_first>

<action>
1. 在 `storage.service.ts` 中添加方法:
   - `generateWatermarkedImage(imageUrl: string, watermarkText: string): Promise<string>`:
     a. 下载原始图片
     b. 使用 sharp 添加水印:
        - 水印文字: watermarkText（默认"寻裳 AI 试衣"）
        - 位置: 右下角，距边缘 20px
        - 样式: 白色半透明文字，font-size 24px，opacity 0.6
        - 使用 SVG 文字渲染 + composite 叠加
     c. 上传水印图到 MinIO，路径 `tryon-results/watermarked/`
     d. 返回水印图 URL

   - `getCDNUrl(path: string): string`:
     a. 读取 `CDN_BASE_URL` 环境变量
     b. 如果配置了 CDN，返回 `CDN_BASE_URL/{path}`
     c. 否则返回原始 MinIO URL

2. 在 `try-on.service.ts` 中:
   - 试衣完成后，自动生成水印版本:
     ```typescript
     const watermarkedUrl = await this.storage.generateWatermarkedImage(
       result.resultImageUrl,
       '寻裳 AI 试衣'
     );
     ```
   - 在 VirtualTryOn 记录中保存 watermarkedUrl（需在 schema 中添加字段）
</action>

- storage.service.ts 包含 getCDNUrl 方法

### Task 3: ML 服务虚拟试衣路由更新

<read_first>
- ml/services/visual_outfit_service.py
- ml/api/main.py
- ml/api/config.py
</read_first>

<action>
1. 在 `ml/api/config.py` 中添加配置:
   ```python
   DOUBAO_SEEDREAM_API_KEY: str = ""
   DOUBAO_SEEDREAM_API_URL: str = "https://visual.volcengineapi.com/v1/aigc/generate"
   DOUBAO_SEEDREAM_MODEL: str = "doubao-seedream-3-0-t2i-250415"
   ```

2. 创建 `ml/services/virtual_tryon_service.py`:
   - `VirtualTryonService` 类:
     - `__init__`: 读取 Doubao-Seedream 和 GLM API 配置
     - `async generate_tryon(person_image: str, garment_image: str, category: str, prompt: str) -> dict`:
       a. 尝试 Doubao-Seedream API:
          - 构建 req_key="high_aes_general_v21_L" 请求
          - 异步轮询结果
       b. 失败时降级到 GLM-4V:
          - 多模态请求生成换装描述
       c. 返回 `{ success: bool, result_url: str, provider: str, processing_time: float }`
     - `async _call_doubao_seedream(person_b64, garment_b64, prompt) -> dict`
     - `async _call_glm_fallback(person_b64, garment_b64, prompt) -> dict`
     - `async _poll_seedream_result(task_id) -> dict`

3. 创建 `ml/api/routes/virtual_tryon.py`:
   - `POST /api/v1/virtual-tryon/generate`:
     - 接受: person_image_url, garment_image_url, category, prompt
     - 调用 VirtualTryonService.generate_tryon
     - 返回结果
   - `GET /api/v1/virtual-tryon/health`:
     - 检查 API 可用性

4. 在 `ml/api/main.py` 中注册路由:
   ```python
   try:
       from ml.api.routes.virtual_tryon import router as virtual_tryon_router
       app.include_router(virtual_tryon_router)
   except Exception as e:
       logging.getLogger(__name__).warning("Failed to load virtual tryon routes: %s", e)
   ```

5. 更新 `visual_outfit_service.py`:
   - 将 `_generate_virtual_tryon` 方法中的 IDM-VTON URL 替换为新的 `/api/v1/virtual-tryon/generate`
</action>
