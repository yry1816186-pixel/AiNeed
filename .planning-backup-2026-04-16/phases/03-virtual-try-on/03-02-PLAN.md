# Plan 02: 后端试衣服务增强 — 照片修复/进度/重试/持久化

## Objective
增强后端试衣服务：添加照片自动修复、WebSocket 实时进度推送、每日重试限制、结果持久化与分类查询。

## Tasks

### Task 1: 添加每日重试限制

<read_first>
- apps/backend/src/modules/try-on/try-on.service.ts
- apps/backend/src/modules/try-on/try-on.controller.ts
- apps/backend/src/modules/security/rate-limit/ai-quota.guard.ts
</read_first>

<action>
1. 在 `try-on.service.ts` 中添加每日重试限制逻辑:
   - 添加 `checkDailyRetryLimit(userId: string): Promise<void>` 方法
   - 使用 Redis key `tryon:daily:{userId}:{YYYYMMDD}` 作为计数器
   - TTL 设为到当天结束（计算剩余秒数）
   - 限制: 每日 3 次免费重试
   - 超出限制时抛出 `BadRequestException('今日免费试衣次数已用完，明天再来吧')`

2. 在 `createTryOnRequest` 方法中:
   - 在创建新请求前调用 `checkDailyRetryLimit(userId)`
   - 请求创建成功后递增 Redis 计数器: `INCR tryon:daily:{userId}:{YYYYMMDD}`

3. 添加 `getDailyRetryCount(userId: string): Promise<number>` 方法:
   - 读取 Redis 计数器
   - 用于前端显示剩余次数

4. 在 `try-on.controller.ts` 中添加端点:
   - `GET /try-on/daily-quota` 返回 `{ used: number, limit: number, remaining: number }`
</action>

<acceptance_criteria>
- try-on.service.ts 包含 checkDailyRetryLimit 方法
- 使用 Redis key `tryon:daily:{userId}:{YYYYMMDD}` 作为计数器
- 每日限制 3 次
- createTryOnRequest 在创建前检查限制
- GET /try-on/daily-quota 端点存在
- TypeScript 编译无错误
</acceptance_criteria>

- apps/backend/src/common/gateway/notification.service.ts
- apps/backend/src/modules/queue/queue.processor.ts

<action>
1. 在 `notification.service.ts` 中添加方法:
   ```typescript
   async notifyTryOnProgress(userId: string, tryOnId: string, progress: number, stage: string): Promise<void>`
   ```
   - 通过 WebSocket 发送 `try_on_progress` 事件到 `user:{userId}` room
   - payload: `{ tryOnId, progress, stage, timestamp }`
   - stage 值: "uploading" | "processing" | "generating" | "completed" | "failed"

2. 在 `notification.gateway.ts` 中添加 `try_on_progress` 事件类型到 NotificationPayload.type 联合类型

3. 在 `queue.processor.ts` 的 VirtualTryOnProcessor.process 方法中:
   - 在关键节点调用 `notifyTryOnProgress`:
     - 开始处理: progress=10, stage="processing"
     - API 调用前: progress=30, stage="generating"
     - API 返回后: progress=70, stage="generating"
     - 上传结果: progress=90, stage="processing"
     - 完成: progress=100, stage="completed"
</action>

<acceptance_criteria>
- notification.service.ts 包含 notifyTryOnProgress 方法
- 发送 `try_on_progress` WebSocket 事件
- payload 包含 tryOnId, progress, stage, timestamp

- TypeScript 编译无错误
</acceptance_criteria>

### Task 5: 照片自动修复集成

<read_first>
- apps/backend/src/modules/try-on/try-on.service.ts
- ml/api/routes/photo_quality.py
- ml/services/photo_quality_analyzer.py
</read_first>

<action>
1. 在 `try-on.service.ts` 中添加照片质量检查和自动修复:
   - 注入 ConfigService 读取 `TRYON_AUTO_ENHANCE` 环境变量（默认 true）
   - 在 `createTryOnRequest` 中，照片验证后:
     a. 调用 ML 服务 `/api/v1/photo-quality/analyze` 检查照片质量
     b. 如果质量分低于 0.6，调用 ML 服务 `/api/v1/photo-quality/enhance` 增强处理
     c. 增强后的照片 URL 替换原始 URL 用于试衣生成
   - 添加 `checkAndEnhancePhoto(photoUrl: string): Promise<string>` 私有方法

2. 在 ML 服务 `photo_quality.py` 中添加 `/api/v1/photo-quality/enhance` 端点:
   - 接受图片 URL 或 base64
   - 使用 Pillow 进行: 亮度归一化、对比度增强、锐化、背景简化
   - 返回增强后的图片 URL（上传到 MinIO）
</action>

<acceptance_criteria>
- try-on.service.ts 包含 checkAndEnhancePhoto 方法
- 照片质量低于阈值时自动增强
- ML 服务包含 /api/v1/photo-quality/enhance 端点
- TypeScript + Python 编译无错误
</acceptance_criteria>
