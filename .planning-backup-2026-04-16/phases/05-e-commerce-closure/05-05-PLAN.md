Add the following methods to `apps/mobile/src/services/api/commerce.api.ts`, following existing patterns (apiClient calls + normalize functions):

**Payment APIs:**
- `createPayment(orderId: string, provider: 'alipay' | 'wechat')` -- POST /api/v1/payment/create
- `pollPaymentStatus(orderId: string)` -- GET /api/v1/payment/status/:orderId

**Coupon APIs:**
- `validateCoupon(code: string, orderAmount: number, categoryIds?: string[], brandId?: string)` -- POST /api/v1/coupons/validate
- `applyCoupon(code: string)` -- POST /api/v1/coupons/apply
- `getUserCoupons(status?: string)` -- GET /api/v1/coupons
- `getApplicableCoupons()` -- GET /api/v1/coupons/applicable

**Refund APIs:**
- `createRefund(data: { orderId, type, reason, description?, images? })` -- POST /api/v1/refund-requests
- `getOrderRefunds(orderId: string)` -- GET /api/v1/refund-requests/order/:orderId
- `getUserRefunds()` -- GET /api/v1/refund-requests
- `addRefundTracking(id: string, trackingNumber: string)` -- PATCH /api/v1/refund-requests/:id/tracking

**Stock Notification APIs:**
- `subscribeStockNotification(itemId: string, color?: string, size?: string)` -- POST /api/v1/stock-notifications
- `unsubscribeStockNotification(id: string)` -- DELETE /api/v1/stock-notifications/:id
- `getStockNotifications()` -- GET /api/v1/stock-notifications

**Order Enhancement APIs:**
- `confirmReceipt(orderId: string)` -- PATCH /api/v1/orders/:id/confirm
- `softDeleteOrder(orderId: string)` -- DELETE /api/v1/orders/:id
- `getOrdersByTab(tab: string, page?: number, limit?: number)` -- GET /api/v1/orders/tab/:tab

**Size Recommendation APIs:**
- `getSizeRecommendation(itemId: string)` -- GET /api/v1/size-recommendation/:itemId
- `getSizeChart(itemId: string)` -- GET /api/v1/size-recommendation/:itemId/chart

**Search Enhancement APIs:**
- `getFilterOptions(category?: string)` -- GET /api/v1/search/filter-options

**Clothing Enhancement APIs:**
- `getRelatedItems(itemId: string)` -- GET /api/v1/clothing/:id/related
- `getSubcategories(category?: string)` -- GET /api/v1/clothing/subcategories

**Merchant APIs:**
- `applyForMerchant(data: { brandName, businessLicense, contactName, phone, description })` -- POST /api/v1/merchants/apply
- `getMerchantApplicationStatus()` -- GET /api/v1/merchants/application
- `getMerchantPendingApplications()` -- GET /api/v1/merchants/applications

**Cart Enhancement APIs:**
- `getCartSummary(couponCode?: string)` -- GET /api/v1/cart/summary with couponCode param
- `getInvalidCartItems()` -- GET /api/v1/cart/invalid
- `batchDeleteCartItems(ids: string[])` -- DELETE /api/v1/cart/batch
- `moveCartToFavorites(ids: string[])` -- POST /api/v1/cart/move-to-favorites
- `updateCartItemSku(id: string, color?: string, size?: string)` -- PATCH /api/v1/cart/:id/sku

Add normalize functions for new response types:
- normalizeCoupon, normalizeRefundRequest, normalizeStockNotification
- normalizeSizeRecommendation, normalizeFilterOptions, normalizeSubcategory

Add TypeScript types for new entities at top of file or in a separate types file:
