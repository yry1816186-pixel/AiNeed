## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client -> Coupon API | Untrusted coupon codes, user-supplied amounts |
| client -> Stock Notification API | Untrusted item IDs, potential subscription spam |
| admin -> Coupon CRUD | Trusted but needs auth verification |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | S (Spoofing) | Coupon validation | mitigate | Verify userId from JWT, not request body |
| T-05-02 | T (Tampering) | Coupon discount calculation | mitigate | All discount calculation server-side, never trust client amounts |
| T-05-03 | R (Repudiation) | Coupon usage tracking | mitigate | UserCoupon records with usedAt/orderId for audit trail |
| T-05-04 | I (Info Disclosure) | Stock notification | accept | No PII in subscription data, only userId+itemId |
| T-05-05 | D (Denial of Service) | Coupon brute-force | mitigate | 8+ char random codes, rate limit validation endpoint |
| T-05-06 | E (Elevation) | Stock notification spam | mitigate | Unique constraint per userId+itemId+color+size |
</threat_model>
