### Task 2: Privacy Consent Backend Integration
<task type="auto">
<read_first>
apps/backend/src/modules/privacy/privacy.service.ts
apps/backend/src/modules/privacy/privacy.controller.ts
apps/backend/src/modules/auth/auth.service.ts
</read_first>
<action>
Enhance privacy consent recording to automatically capture consent on registration and support version tracking:

1. Check if `apps/backend/src/modules/privacy/privacy.controller.ts` has consent recording endpoint. If not, add:
   - `POST /privacy/consent` - Record user consent (called from mobile during registration)
   - `GET /privacy/consent` - Get user's consent history
   - `GET /privacy/policy-version` - Get current policy version
   - `POST /privacy/consent/reconfirm` - Re-confirm consent after policy update

2. Update `apps/backend/src/modules/auth/auth.service.ts`:
   - In the registration flow, after user creation, call PrivacyService.recordConsent() for both:
     - consentType: "terms_of_service" with version from env/config
     - consentType: "privacy_policy" with version from env/config
   - This ensures consent is captured server-side even if mobile client fails to record it

3. Add a policy version constant (read from env or hardcoded as "1.0.0"):
   - Create `apps/backend/src/modules/privacy/privacy-version.ts`:
     ```typescript
     export const CURRENT_TERMS_VERSION = '1.0.0';
     export const CURRENT_PRIVACY_VERSION = '1.0.0';
     export const POLICY_UPDATED_AT = '2026-04-14';
     ```

4. Ensure privacy module is imported in auth module (or use forwardRef if circular dependency)
</action>
<acceptance_criteria>
- Registration automatically records privacy and terms consent
- Consent version tracking in place
- Privacy controller has consent endpoints
- Policy version centralized constant
</acceptance_criteria>
</task>
