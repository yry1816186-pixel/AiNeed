# Plan 07-02: Brand QR Code System + Brand Portal Backend

## Objective

Build the brand collaboration system: QR code generation with embedded product data, scan-to-import flow, brand management portal backend with product data management, scan statistics, and user preference analytics.

## Context

@07-CONTEXT.md -- decisions D-11 through D-17
@apps/backend/src/modules/brands/brands.service.ts -- existing brand queries
@apps/backend/src/modules/merchant/merchant.service.ts -- merchant data dashboard
@apps/backend/prisma/schema.prisma -- Brand, BrandMerchant models

## Threat Model

| File | Surface | Mitigation |
|------|---------|------------|
| brand-portal.controller.ts | brand data access | merchant auth guard, scope to own brandId |
| qr-code.dto.ts | QR data injection | validate brand ownership, sanitize input |
| brands.controller.ts | public scan endpoint | rate limit, no auth required for scan |

<task type="auto">
<read_first>

<action>
1. Add new models to schema.prisma:

   - `generateQRCode(brandId, productId, productData)` -- create BrandQRCode with encoded payload (brand name, product name, sku, color, size, material, price). </action>

### Task 3: Add Scan + QR Code Endpoints to BrandsController

<task type="auto">
<read_first>
- apps/backend/src/modules/brands/brands.controller.ts
</read_first>
<action>
1. Add new endpoints to `BrandsController`:
   - `POST /brands/qr-codes` (MerchantAuthGuard) -- generate QR code for brand's product
   - `GET /brands/qr-codes` (MerchantAuthGuard) -- list brand's QR codes
   - `GET /brands/qr-codes/:code` (public, no auth) -- scan/resolve QR code, returns product data
   - `POST /brands/qr-codes/:code/scan` (optional auth) -- record scan event + import to wardrobe if authenticated
   - `PATCH /brands/qr-codes/:id/deactivate` (MerchantAuthGuard) -- disable QR code

2. The scan endpoint (`GET /brands/qr-codes/:code`):
   - Look up QR code by code
   - Return brand name, product details, suggested match
   - This is the endpoint mobile app calls when scanning a brand QR code

3. The import endpoint (`POST /brands/qr-codes/:code/scan`):
   - Record the scan event
   - If userId provided, add product to user's wardrobe
   - Return success with wardrobe entry

4. Add Swagger decorators
</action>
<acceptance_criteria>
- POST /brands/qr-codes creates QR code (requires merchant auth)
- GET /brands/qr-codes/:code returns product data (public, no auth)
- POST /brands/qr-codes/:code/scan records scan and imports to wardrobe
- PATCH deactivate sets isActive=false
- Rate limiting considered for public scan endpoint
</acceptance_criteria>
</task>
