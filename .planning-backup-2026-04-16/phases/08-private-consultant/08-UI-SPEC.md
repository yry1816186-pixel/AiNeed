## Interaction Patterns

### Smart Matching Flow (ADV-01, ADV-03, D-01/02/03)

1. User taps "提交需求" on AdvisorList
2. Bottom sheet appears: service type chips + budget range slider + notes input + location toggle
3. User submits → loading state with "正在为你智能匹配..." animation
4. Match results appear as sorted list (highest match% first)
5. Each card shows: match% badge (accent) + match reasons (e.g. "擅长你的体型", "评分最高", "距离最近")
6. User taps card → navigate to AdvisorProfile

### Booking Flow (ADV-04, ADV-07, D-09/10/11, CC-37)

1. User taps "预约顾问" on AdvisorProfile → navigate to Booking
2. Select service type → calendar highlights available dates
3. Select date → time slot list appears below calendar
4. Select time slot → slot highlights (accent border)
5. Add notes (optional) → price summary updates
6. Tap "支付定金 ¥{amount}" → PaymentModule handles payment
7. Payment success → booking confirmed, system message in chat, push notification

### Chat Flow (ADV-05, ADV-06, D-04/05/06/07/08)

1. After booking confirmed → chat room auto-created → navigate to Chat
2. WebSocket connects to /ws/chat namespace with JWT auth
3. User sends text → ChatBubble appears with user variant (coral gradient)
4. Consultant replies → ChatBubble appears with consultant variant (sage tint)
5. Typing indicator shows when consultant is typing
6. Read receipts update in real-time
7. Consultant shares proposal → ProposalCard appears in chat
8. User taps "保存到灵感衣橱" → saved to wardrobe

### Review Flow (ADV-09, D-12/13)

1. Service completed → user receives notification to review
2. Review bottom sheet: Rating (interactive, large) + tag chips (专业/耐心/有创意/准时/沟通顺畅) + text input + before/after image upload
3. Submit review → ConsultantProfile.rating and reviewCount update → ranking recalculated

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| Touch targets | Minimum 44px (exception to 4px grid) |
| Color contrast | Accent #C67B5C on white = 3.2:1 (AA for large text), use #A85A3A for small text (4.6:1 AA) |
| Screen reader | All images have accessibilityLabel, icon-only buttons: share="分享顾问", image picker="选择图片", download="下载文件", ChatBubble has role="link" for proposals |
| Focus order | Calendar → time slots → notes → CTA (logical top-to-bottom) |
| Error announcements | Toast with accessibilityLiveRegion="assertive" |
