# 新功能需求 - 已决策 (v3.3)

## Feature 1: Q版个人形象 (Q-Avatar) - 已决策

**核心需求**:
- 根据用户数据(填写信息)生成个人专属Q版形象
- 形象必须**会动**(Skia本地动画)，提供情绪价值
- 审美要求极高，不能有AI痕迹，不能丑

### 已做出决策

| 问题 | 决策 | 理由 |
|------|------|------|
| 画风 | **Q版潮玩风 (POP MART风格矢量版)** | 大头、圆润、精致质感，国内用户熟悉且有情感连接 |
| 技术路径 | **react-native-skia动态绘制** | 客户端实时渲染，零服务端成本，零外包 |
| 生成方式 | **用户选择模板+手动调参** | 选择脸型/眼型/肤色/发型等参数，无需AI |
| 动画范围 | 待机/开心/挥手/点头/思考/比心/换装过渡 | 7种本地Skia属性动画 |
| 与虚拟试衣整合 | **简化映射(颜色+类型)** | 换装结果→Q版形象同步更新颜色/类型，非精确UV纹理 |
| 与服装定制整合 | **简化映射(图案色块)** | 定制服装图案→Q版形象叠加简化色块 |

### 技术架构

```
用户选择模板 → 调整参数(脸型/眼型/肤色/发型)
    → react-native-skia 实时绘制
    → 本地动画(useFrameCallback)
    → 换装时更新 clothing_map {slot: {color, type}}
    → 缩略图截图上传到MinIO
```

### 关键技术选型

| 环节 | 技术 | 成本 |
|------|------|------|
| 形象渲染 | react-native-skia (Shopify) | 免费 |
| 动画系统 | Skia + useFrameCallback | 免费 |
| 参数存储 | 后端JSONB (avatar_params) | 已有 |
| 服装映射 | clothing_map JSONB (颜色+类型) | 已有 |
| 缩略图 | 客户端截图+上传MinIO | 已有 |

### 新增数据库表
- avatar_templates (Q版形象模板+Skia绘制配置)
- user_avatars (用户形象参数+服装映射)

### 新增API端点
- GET /avatar/templates, POST /avatar/create, GET /avatar/me
- PATCH /avatar/me, POST /avatar/me/dress, POST /avatar/me/thumbnail

## Feature 2: 个人服装定制 (Clothing Customization) - 已决策

**核心需求**:
- 用户上传图案/设计 -> 选择衣服类型和面料 -> 排列图案 -> 预览效果
- 定制服装可映射到Q版形象(简化颜色+类型)
- 满意后下单，通过EPROLO POD生产
- 不限于衣服：卫衣/帽子/包包/手机壳

### 已做出决策

| 问题 | 决策 | 理由 |
|------|------|------|
| 优先级 | **核心功能，必须做** | 差异化护城河，商业价值高 |
| 图案编辑器 | **WebView+Canvas编辑器** | 缩放/旋转/拖动/平铺/文字/滤镜 |
| 预览方式 | **2D模板预览** | 图案映射到产品模板2D图，非3D |
| 订单生产 | **EPROLO POD** | API公开(eprolo.com/eprolo-api)，1500+产品，一件起订 |
| 设计市集 | **社区+市集** | 用户可发布设计、其他用户购买，设计师分成 |

### 商业模式

```
定制价格 = 成本 + 平台溢价
    T恤: 成本~30 -> 售价89-149 -> 毛利~60-120
    卫衣: 成本~60 -> 售价159-249 -> 毛利~100-190
    帽子: 成本~20 -> 售价59-89 -> 毛利~40-70
    包包: 成本~35 -> 售价99-179 -> 毛利~65-145

设计市集分成: 设计师60% / 平台30% / 生产方10%
```

### POD服务商: EPROLO (已确认)
- API文档: eprolo.com/eprolo-api
- 产品线: 1500+产品(T恤/卫衣/帽子/包包/手机壳/鞋子等)
- 起订量: 一件起订
- 生产周期: 3-7天
- 面向: 全球市场(中国制造+全球物流)

### 新增数据库表
- custom_designs (定制设计)
- custom_orders (定制订单)
- product_templates (产品模板)
- design_likes (设计点赞)

### 新增API端点
- GET/POST/PATCH/DELETE /customize/designs
- POST /customize/designs/:id/preview, /customize/designs/:id/publish
- POST/GET/PATCH /custom-orders
- GET /market/designs, POST /market/designs/:id/like

## Feature 3: VTO技术路线 - 已决策

### 已做出决策

| 问题 | 决策 | 理由 |
|------|------|------|
| 自训练 vs 云端API | **MVP用云端API** | 先验证需求，日均>100次再考虑自部署 |
| 主供应商 | **阿里云百炼 OutfitAnyone** | ~0.50元/张，中国市场最佳，支持上下装组合 |
| 备选供应商 | **Kolors via fal.ai** | $0.07/张，快手模型，亚洲时尚数据好 |
| 多件试穿 | **分步合成** | 上衣->下装->组合->叠加配饰，无API支持5-6件同时试穿 |
| Q版形象换装 | **Skia简化映射** | 颜色+类型映射，非精确纹理替换 |
| GPU成本 | **0** | 纯云端API调用，RTX 4060留待后续微调 |

### 技术降级方案

| 功能 | 主方案 | 降级方案 | 触发条件 |
|------|--------|---------|---------|
| 虚拟试衣(真实照片) | 阿里云百炼OutfitAnyone | Kolors via fal.ai | 主API不可用 |
| Q版形象渲染 | react-native-skia | 静态SVG图片 | Skia兼容性问题 |
| AI造型师 | GLM-5 Agent模式 | DeepSeek API | GLM-5故障 |
| 全文搜索 | Elasticsearch | PostgreSQL tsvector | ES资源不足 |
| 向量搜索 | Qdrant | pgvector | Qdrant不可用 |
| 知识图谱 | Neo4j | PostgreSQL JSONB | Neo4j复杂度过高 |

## 架构影响总结

**被移除的组件**:
- AnyDressing自部署(A100 GPU) -> 月省~8-12K
- SMPLify-X 3D人体重建 -> 不再需要
- YOLOv8-pose 人体检测 -> 不再需要
- MediaPipe Face Mesh -> 不再需要(简化为手动选参)
- Spine 2D / Lottie -> 不再需要(Skia替代)
- Blender -> 不再需要(无需3D建模和预渲染)
- HICUSTOM POD -> EPROLO替代

**新增的组件**:
- react-native-skia (Q版形象渲染+动画)
- 阿里云百炼 OutfitAnyone API (真实VTO)
- EPROLO POD API (定制生产)
- Canvas/WebGL图案编辑器

**成本变化**:
- 月运营: ~9,200-15,100 -> ~960-3,100 (降低85%，详见宪法第七节)
- 一次性开发: ~5,000-15,500(3D模板) -> ~1,500-2,500(标注+规则) (降低80%)

**开发Session**:
- AV1: Q版形象管理服务
- AV2: 形象模板管理
- AV3: Q版形象页面(创建/编辑/展示)
- AV4: Skia渲染组件+本地动画
- CU1: 图案编辑器组件
- CU2: 定制预览页面
- CU3: 定制订单管理(EPROLO对接)
- CU4: 设计市集

## Feature 4: 前端设计系统 - 已决策

### 已做出决策

| 问题 | 决策 | 理由 |
|------|------|------|
| 色彩体系 | **白底+深色强调+玫瑰红点缀** | 得物/小红书风格，干净明亮，时尚感强 |
| 首页布局 | **Q版形象为核心** | 情绪价值+视觉焦点+差异化 |
| AI造型师交互 | **卡片交互式(非纯对话)** | 选场合/预算/风格→生成搭配卡片，更直观 |
| 衣橱页面 | **Q版形象可视化** | 点击形象部位管理服装，创新交互 |
| 个人页 | **经典个人中心** | 数据卡片+功能菜单，信息清晰 |
| 底部导航 | **5 Tab + 中间"+"大按钮** | 首页/AI造型/+操作/衣橱/我的 |
| 图标系统 | **SVG精致图标** | 不使用emoji，专业感 |
| 定制页面 | **5步向导式** | 选产品→上传图案→编辑布局→预览→下单 |
| 设计市集 | **双列瀑布流** | 类似小红书，预览图+设计师+价格 |
| 社区风格 | **小红书风格** | 双列Feed+发帖/评论/点赞/关注 |
| 虚拟试衣入口 | **双入口** | 服装详情页+个人页均可发起 |
