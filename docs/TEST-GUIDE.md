# AiNeed 功能测试指南

## 📋 测试前准备

### 1. 启动后端服务

#### 启动主API服务
```bash
cd C:/AiNeed/apps/backend
npm install
npm run dev
```
服务地址：http://localhost:3001

#### 启动AI服务
```bash
cd C:/AiNeed/apps/backend
npm run ai-service
```
服务地址：http://localhost:8001

### 2. 配置API地址

#### 对于Android模拟器
应用默认配置为：`http://10.0.2.2:3001/api/v1`
这是模拟器访问主机的特殊地址，无需修改。

#### 对于真机测试
1. 获取电脑IP地址：
```bash
# Windows
ipconfig

# 查找 IPv4 地址，例如：192.168.1.100
```

2. 修改配置文件 `apps/mobile/app.json`：
```json
{
  "extra": {
    "API_URL": "http://192.168.1.100:3001/api/v1",
    "AI_SERVICE_URL": "http://192.168.1.100:8001"
  }
}
```

3. 重新编译APK

### 3. 启动模拟器或连接设备

```bash
# 查看连接的设备
adb devices

# 启动模拟器（如果需要）
emulator -avd <avd_name>
```

## 🔍 调试技巧

### 查看应用日志
```bash
# 实时查看日志
adb logcat | grep -i "aineed"

# 保存日志到文件
adb logcat > aineed_log.txt
```

### 查看网络请求
```bash
# 查看网络日志
adb logcat | grep -i "network"
```

### 清除应用数据
```bash
# 清除应用数据
adb shell pm clear com.aineed.app
```

### 重新安装
```bash
# 卸载应用
adb uninstall com.aineed.app

# 重新安装
adb install app-release.apk
```
