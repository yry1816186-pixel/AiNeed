# Plan 08-05: 删除重复类型 + 全量编译验证

<objective>
删除 mobile `src/shared/types/` 副本目录，清理所有 deprecated 别名，执行全量编译验证，确保类型系统单一真相源建立完成。
</objective>

<threat_model>
**Threat:** 删除 src/shared/types/ 可能破坏引用该路径的 import
**Mitigation:** 先搜索所有引用，确保无直接 import 该路径的代码
**Severity:** Medium
</threat_model>

<tasks>

### Task 8-05-01: 搜索并修复 src/shared/types/ 引用

<read_first>
- 搜索 `apps/mobile/src/` 中所有引用 `shared/types/` 的 import 语句
</read_first>

<action>
1. 全局搜索 mobile 中引用 `shared/types` 的代码：
   ```
   grep -r "from.*shared/types" apps/mobile/src/ --include="*.ts" --include="*.tsx"
   grep -r "from.*\.\./shared/types" apps/mobile/src/ --include="*.ts" --include="*.tsx"
   ```
2. 将所有引用改为从 `@xuno/types` 或 `src/types/` 导入
3. 验证无残留引用
</action>

<acceptance_criteria>
- `grep -r "shared/types" apps/mobile/src/ --include="*.ts" --include="*.tsx"` 返回 0 结果
- 所有类型导入指向 `@xuno/types` 或 `src/types/`
</acceptance_criteria>

### Task 8-05-03: 清理 deprecated 别名

<read_first>
- packages/types/src/clothing.ts
- apps/mobile/src/types/clothing.ts
</read_first>

<action>
1. 搜索所有 `@deprecated` 类型别名：
   - `ClothingItem = CatalogItem` (在 @xuno/types)
   - `ClothingItem = WardrobeItem` (在 mobile)
2. 评估是否可以安全移除：
   - 如果有外部消费者仍使用 `ClothingItem`，保留 deprecated 别名

3. 搜索所有 `ClothingItem` 引用，确认已全部迁移
4. 如果安全，删除 deprecated 别名
</action>

<acceptance_criteria>
- 所有 `ClothingItem` 引用已迁移为 `CatalogItem` 或 `WardrobeItem`

### Task 8-05-04: 全量编译验证

<read_first>
- 全项目
</read_first>

<action>
1. 构建共享包：
   ```bash
   cd packages/types && pnpm build
   ```

2. 验证后端编译：
   ```bash
   cd apps/backend && npx tsc --noEmit
   ```

3. 验证移动端编译：
   ```bash
   cd apps/mobile && npx tsc --noEmit
   ```

4. 验证无残留冲突：
   ```bash
   # 无 enum 定义
   grep -r "^export enum " packages/types/src/ --include="*.ts"
   # 无 SharedXxx 别名
   grep -r "SharedGender\|SharedBodyType\|SharedSkinTone\|SharedClothingItem\|SharedClothingCategory" apps/mobile/src/ --include="*.ts"
   # 无 Date 类型
   grep -r ": Date" packages/types/src/ --include="*.ts"
   # 无 @xuno/shared 引用
   grep -r "@xuno/shared" apps/ packages/ --include="*.ts" --include="*.tsx"
   # 无 shared/types 副本
   test -d apps/mobile/src/shared/types
   ```

5. 验证类型导出完整性：
   ```bash
   # @xuno/types 导出的类型数量
   cd packages/types && node -e "const t = require('./dist/index.js'); console.log(Object.keys(t).length)"
   ```

6. 运行后端测试（确保无回归）：
   ```bash
   cd apps/backend && npx jest --passWithNoTests
   ```
</action>

<acceptance_criteria>
- `packages/types` 构建成功
- 后端 `tsc --noEmit` 通过
- 移动端 `tsc --noEmit` 通过
- 无 enum 定义残留
- 无 SharedXxx 别名残留
- 无 Date 类型残留
- 无 @xuno/shared 引用
- `src/shared/types/` 已删除
- 后端测试通过
</acceptance_criteria>
