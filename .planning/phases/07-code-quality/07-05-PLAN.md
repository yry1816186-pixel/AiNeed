# Plan 07-05: 导航类型系统修复 — 消除所有 `as never` 和 `as any` 导航

<objective>
修复移动端导航类型系统，消除所有 `as never`（51处）和 `as any`（12处）跨 Stack 导航断言，建立类型安全的导航架构。核心策略：统一 RootStackParamList 为层级结构，每个 Screen 使用自身 Stack 的 ScreenProps，跨 Stack 导航使用 navigationService。
</objective>

<threat_model>
**Threat:** 类型不安全的导航可能在运行时导航到不存在的路由，导致白屏或崩溃
**Mitigation:** 通过 TypeScript 编译时检查确保所有导航目标路由存在且参数正确
</threat_model>

### 10.1 修复 expo-router polyfill

The polyfill uses `navigation.navigate(path as never)` and `navigation.navigate(path as any, params as any)`. Since this is a dynamic path-based router that maps string paths to routes, it cannot be fully type-safe by design. The fix is to use a more explicit type assertion with a comment:

Replace:

```typescript
navigation.navigate(path as never);
```

with:

```typescript
// Dynamic path routing — cannot be statically typed
(navigation.navigate as (name: string, params?: object) => void)(path);
```

Replace:

```typescript
navigation.navigate(path as any, params as any);
```

with:

```typescript
// Dynamic path routing — cannot be statically typed
(navigation.navigate as (name: string, params?: object) => void)(path, params);
```

And on line 129:

```typescript
navigation.navigate(path as never);
```

with:

```typescript
(navigation.navigate as (name: string, params?: object) => void)(path);
```
