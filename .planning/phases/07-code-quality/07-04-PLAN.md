# Plan 07-04: 移动端测试覆盖率提升

<objective>
为移动端关键 Store、API 服务和组件补充测试，将测试覆盖率从 ~5% 提升至 30%+，并将 Jest 覆盖率阈值从不现实的 60% 调整为 30%。
</objective>

<threat_model>
**Threat:** 低测试覆盖率意味着 UI 交互和状态管理可能存在未发现的 bug
**Mitigation:** 优先为涉及购物流程和 AI 交互的模块补充测试
**Severity:** Medium
</threat_model>
