# Plan 07-03: 后端测试覆盖率提升

<objective>
为后端无测试的关键业务模块补充测试，将测试覆盖率从 ~15% 提升至 50%+，并将 Jest 覆盖率阈值从 20% 提升至 50%。
</objective>

<threat_model>
**Threat:** 低测试覆盖率意味着关键业务路径（支付、退款、优惠券）可能存在未发现的 bug
**Mitigation:** 优先为涉及金钱交易的模块补充集成测试
**Severity:** High
</threat_model>

<tasks>
