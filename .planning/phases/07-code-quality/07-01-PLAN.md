# Plan 07-01: 后端 `any` 类型消除

<objective>
消除后端生产代码中的 `any` 类型，从 ~39 处降至 < 10 处（仅保留 Prisma 层豁免），并将 ESLint `no-explicit-any` 从 "warn" 升级为 "error"。

**口径说明：** REQUIREMENTS.md 中 QUAL-02 记录的 ~668 处 `any` 包含所有文件（含 spec 测试文件、配置文件、注释中的 `any` 引用等）。本计划统计口径为生产代码中的显式 `: any`、`as any`、`any[]` 模式，实际约 39 处。修复后生产代码 < 10 处，加上测试文件修复，总 `any` 数量将大幅降低。
</objective>

<threat_model>
**Threat:** `request: any` 允许未经验证的属性访问，可能导致权限绕过
**Mitigation:** 替换为 Express `Request` 类型，编译时捕获非法属性访问
**Severity:** Medium
</threat_model>

<tasks>

### Task 7-01-01: 修复简单 `any` 模式

<read_first>
- apps/backend/src/common/interceptors/sensitive-data.interceptor.ts
- apps/backend/src/common/decorators/api-paginated.ts
- apps/backend/src/modules/merchant/guards/merchant-auth.guard.ts
- apps/backend/src/modules/feature-flags/feature-flag.controller.ts
- apps/backend/src/modules/feature-flags/feature-flag.service.ts
- apps/backend/src/modules/queue/queue.service.ts
- apps/backend/src/modules/customization/customization.service.ts
</read_first>

<action>
1. `sensitive-data.interceptor.ts`: `Observable<any>` → `Observable<unknown>`, `sanitizeResponse(data: any): any` → `sanitizeResponse(data: unknown): unknown`
2. `api-paginated.ts`: `() => new (...args: any[]) => any` → 添加泛型 `<T extends abstract new (...args: any[]) => any>`
3. `merchant-auth.guard.ts`: `request: any` → `request: Request`, 添加 `import { Request } from 'express'`
4. `feature-flag.controller.ts`: `req: any` → `req: Request`, 添加 `import { Request } from 'express'`
5. `feature-flag.service.ts`: `Map<string, { value: any; ... }>` → `Map<string, { value: string | boolean | number; ... }>`
6. `queue.service.ts`: `status as any` → `status as JobStatus`
7. `customization.service.ts`: `layer.type as any` → `layer.type as CustomizationLayerType`
</action>

<acceptance_criteria>
- `sensitive-data.interceptor.ts` 不包含 `: any` 或 `: Observable<any>`
- `merchant-auth.guard.ts` 包含 `import { Request } from 'express'`
- `feature-flag.controller.ts` 包含 `import { Request } from 'express'`
- `feature-flag.service.ts` 中 Map 不包含 `value: any`
- `queue.service.ts` 不包含 `status as any`
- `customization.service.ts` 不包含 `layer.type as any`
- `cd apps/backend && npx tsc --noEmit` 通过
- `cd apps/backend && npx jest --passWithNoTests` 通过
</acceptance_criteria>
