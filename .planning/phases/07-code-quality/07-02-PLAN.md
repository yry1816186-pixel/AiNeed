# Plan 07-02: 移动端 `any` 类型消除

<objective>
消除移动端生产代码中的 `any` 类型，从 ~67 处降至 < 10 处，并将 ESLint `no-explicit-any` 从 "warn" 升级为 "error"。

**口径说明：** REQUIREMENTS.md 中 QUAL-03 记录的 ~121 处 `any` 包含所有文件（含测试文件、polyfills、注释等）。本计划统计口径为生产代码中的显式 `: any`、`as any` 模式，实际约 67 处。修复后生产代码 < 10 处，加上测试文件修复，总 `any` 数量将大幅降低。
</objective>

<threat_model>
**Threat:** `(response.data as any).reason` 绕过类型检查，可能导致未处理字段访问运行时错误
**Mitigation:** 使用 @xuno/types 定义 API 响应类型
**Severity:** Low
</threat_model>

<tasks>

### Task 7-02-01: 修复组件 props/事件中的 `any`

<read_first>
- apps/mobile/src/components/heartrecommend/AnimatedHeartButton.tsx
- apps/mobile/src/components/tryon/TryOnProgress.tsx
- apps/mobile/src/components/aistylist/TypewriterMessage.tsx
- apps/mobile/src/components/social/SocialInteractions.tsx
- apps/mobile/src/components/recommendation/AlgorithmVisualization.tsx
- apps/mobile/src/components/heartrecommend/SwipeCard.tsx
</read_first>

<action>
1. 所有组件中 `style?: any` → `style?: ViewStyle`，添加 `import { ViewStyle } from 'react-native'`
2. 事件处理器 `(e: any)` → `(e: GestureResponderEvent)` 或 `(e: NativeSyntheticEvent<TextInputChangeEventData>)`
3. `Ionicons name={icon as any}` → 定义 `IoniconName` 类型并替换
4. `SwipeCard.tsx` 中 `_index` prop 类型修复
</action>

<acceptance_criteria>
- 所有组件中不包含 `style?: any`
- 所有组件中不包含 `(e: any)` 或 `(event: any)`
- `SwipeCard.tsx` 不包含 `_index` 相关的 TS 错误
- `cd apps/mobile && npx tsc --noEmit` 通过
</acceptance_criteria>

### Task 7-02-02: 修复导航和 API 中的 `any`

<read_first>
- apps/mobile/src/navigation/navigation.d.ts
- apps/mobile/src/screens/AiStylistChatScreen.tsx
- apps/mobile/src/screens/CartScreen.tsx
- apps/mobile/src/screens/CommunityDetailScreen.tsx
- apps/mobile/src/screens/ProfileScreen.tsx
- apps/mobile/src/services/stylist.api.ts
- packages/types/src/index.ts
</read_first>

<action>
1. 扩展 `navigation.d.ts` 路由类型定义，添加 `RootStackParamList` 包含以下路由参数：
   - `Home: undefined`, `Explore: undefined`, `Heart: undefined`, `Cart: undefined`, `Wardrobe: undefined`, `Profile: { userId?: string }`
   - `AiStylistChat: { sessionId?: string }`, `CommunityDetail: { postId: string }`
   - `ProductDetail: { productId: string }`, `TryOnResult: { tryOnId: string }`
   - `Settings: undefined`, `EditProfile: undefined`, `OrderHistory: undefined`
2. 消除所有 `(navigation as any).navigate(...)` → `navigation.navigate('RouteName', params)`
3. 消除所有 `(route.params as any)?.xxx` → `route.params.xxx`
4. 在 `packages/types/src/index.ts` 中添加 API 响应类型：
   - `StylistSessionResponse { id: string; title: string; createdAt: string }`
   - `StylistMessageResponse { id: string; content: string; role: 'user' | 'assistant'; createdAt: string }`
   - `NotificationListResponse { items: NotificationItem[]; total: number }`
   - 消除 API 服务中的 `(response.data as any).xxx` → 使用上述类型
5. 修复 `keyExtractor={(item: any) => ...}` → `keyExtractor={(item: ClothingItem) => ...}` 或对应列表项类型
</action>

<acceptance_criteria>
- `navigation.d.ts` 包含 `RootStackParamList` 类型定义
- `grep -r "navigation as any" apps/mobile/src/` 返回 0 结果
- `grep -r "route.params as any" apps/mobile/src/` 返回 0 结果
- `grep -r "response.data as any" apps/mobile/src/` 返回 0 结果
- `cd apps/mobile && npx tsc --noEmit` 通过
</acceptance_criteria>
