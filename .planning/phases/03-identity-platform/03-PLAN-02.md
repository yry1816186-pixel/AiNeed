# Plan 02: 迁移 platform 层模块

## Objective

将 recommendations, admin, merchant, analytics, notification, feature-flags, health, queue, metrics 九个模块从 `src/modules/` 迁移到 `src/domains/platform/`，解耦 platform → identity 的违规依赖，将 Recommendations 降级为 platform 层共享服务。

## Threat Model

| Threat ID | Description | Severity | Mitigation |
|-----------|-------------|----------|------------|
| T-3-02 | platform 模块迁移后仍引用 identity 域，违反依赖规则 | Medium | 解耦 AnalyticsModule/FeatureFlagModule 对 AuthModule 的直接导入 |
| T-3-03 | Recommendations 降级后 AiStylistModule 无法获取推荐数据 | High | 通过 DI 接口注入，确保 AiStylistModule 仍可访问推荐能力 |
