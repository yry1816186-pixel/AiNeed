# Plan 03: 配置域间依赖规则和可视化

## Objective

安装并配置 eslint-plugin-boundaries 和 dependency-cruiser，建立域间依赖规则，生成依赖可视化图，确保 identity 和 platform 域的架构约束被强制执行。

## Threat Model

| Threat ID | Description | Severity | Mitigation |
|-----------|-------------|----------|------------|
| T-3-04 | 依赖规则过于严格导致正常开发受阻 | Low | 未迁移模块（src/modules/）允许依赖任何域，过渡期宽松 |

## Tasks

### Task 01: 安装并配置 eslint-plugin-boundaries

<read_first>
- apps/backend/.eslintrc.json (当前 ESLint 配置)
- apps/backend/package.json (当前依赖)
- .planning/research/STACK.md (工具推荐)
- .planning/research/ARCHITECTURE.md (目标架构)
</read_first>

<action>
1. 安装依赖：`cd apps/backend && pnpm add -D eslint-plugin-boundaries`
2. 在 `.eslintrc.json` 中添加配置：
   ```json
   {
     "plugins": ["@typescript-eslint", "import", "boundaries"],
     "settings": {
       "boundaries/elements": [
         {
           "type": "identity",
           "pattern": "src/domains/identity/*",
           "mode": "folder"
         },
         {
           "type": "platform",
           "pattern": "src/domains/platform/*",
           "mode": "folder"
         },
         {
           "type": "common",
           "pattern": "src/common/*",
           "mode": "folder"
         },
         {
           "type": "modules",
           "pattern": "src/modules/*",
           "mode": "folder"
         }
       ]
     },
     "rules": {
       "boundaries/element-types": [2, {
         "default": "allow",
         "rules": [
           {
             "from": ["identity"],
             "disallow": ["modules"],
             "message": "Identity domain must not depend on unmigrated modules. Move the dependency to a domain first."
           },
           {
             "from": ["platform"],
             "disallow": ["identity"],
             "message": "Platform layer must not depend on identity domain. Use events or interfaces instead."
           },
           {
             "from": ["common"],
             "disallow": ["identity", "platform", "modules"],
             "message": "Common layer must not depend on any domain module."
           }
         ]
       }]
     }
   }
   ```
3. 运行 `cd apps/backend && npx eslint src/domains --no-eslintrc -c .eslintrc.json` 验证规则生效
4. 确认当前代码无违规（已知例外：admin → community forwardRef 标记为 Phase 4 待处理）
</action>

<acceptance_criteria>
- `eslint-plugin-boundaries` 在 package.json devDependencies 中
- `.eslintrc.json` 包含 boundaries/elements 和 boundaries/element-types 配置
- `cd apps/backend && npx eslint src/domains --no-error-on-unmatched-pattern` 运行无 platform → identity 违规
- common 层无域依赖违规
</acceptance_criteria>

### Task 02: 安装并配置 dependency-cruiser

<read_first>
- apps/backend/package.json (当前依赖)
- .planning/research/STACK.md (工具推荐)
</read_first>

<action>
1. 安装依赖：`cd apps/backend && pnpm add -D dependency-cruiser`
2. 创建 `.dependency-cruiser.js` 配置文件：
   ```javascript
   module.exports = {
     forbidden: [
       {
         name: "no-platform-to-identity",
         comment: "Platform layer must not depend on identity domain",
         severity: "error",
         from: { path: "^src/domains/platform/" },
         to: { path: "^src/domains/identity/" }
       },
       {
         name: "no-common-to-domain",
         comment: "Common layer must not depend on any domain",
         severity: "error",
         from: { path: "^src/common/" },
         to: { path: "^src/domains/" }
       },
       {
         name: "no-identity-to-unmigrated-modules",
         comment: "Identity domain should not depend on unmigrated modules (transitional)",
         severity: "warn",
         from: { path: "^src/domains/identity/" },
         to: { path: "^src/modules/" }
       }
     ],
     options: {
       doNotFollow: {
         path: "node_modules"
       },
       exclude: {
         path: "\\.d\\.ts$"
       },
       tsPreCompilationDeps: true,
       tsConfig: {
         fileName: "./tsconfig.json"
       }
     }
   };
   ```
3. 在 package.json 中添加脚本：
   ```json
   "dep:check": "depcruise src --validate",
   "dep:graph": "depcruise src/domains --output-type dot | dot -T svg > docs/dependency-graph.svg",
   "dep:report": "depcruise src --output-type html > docs/dependency-report.html"
   ```
4. 运行 `cd apps/backend && npx depcruise src/domains --validate` 验证无违规
5. 运行 `cd apps/backend && npx depcruise src/domains --output-type html > docs/dependency-report.html` 生成可视化报告
</action>

<acceptance_criteria>
- `dependency-cruiser` 在 package.json devDependencies 中
- `.dependency-cruiser.js` 配置文件存在
- package.json 包含 `dep:check`, `dep:graph`, `dep:report` 脚本
- `cd apps/backend && npx depcruise src/domains --validate` 运行无 error 级别违规
- `docs/dependency-report.html` 文件存在
</acceptance_criteria>
