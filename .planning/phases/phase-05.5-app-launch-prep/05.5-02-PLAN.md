# Phase 5.5 Plan 02: Mobile Push Notification Integration & Notification Preferences

## Objective

Integrate push notifications on the mobile side, connect the notification settings screen to the backend API, and wire the notification list to use real API data instead of local seed data.

## Context

@05.5-CONTEXT.md
@apps/mobile/src/screens/NotificationsScreen.tsx
@apps/mobile/src/screens/NotificationSettingsScreen.tsx
@apps/mobile/src/services/apiClient.ts

Existing infrastructure:
- NotificationsScreen with local AsyncStorage-based seed data (needs API integration)
- NotificationSettingsScreen with hardcoded local state (needs API integration)
- UnifiedApiClient with JWT auth, token refresh, JSON:API parsing
- Notification navigation routes already registered (Notifications, NotificationSettings)
- Backend notification API: GET/POST/DELETE /notifications, GET/POST /notifications/settings

## Tasks

### Task 1: Notification API Service Layer
<task type="auto">
<read_first>
apps/mobile/src/services/apiClient.ts
apps/mobile/src/services/api/auth.api.ts
</read_first>
<action>
Create the notification API service and push notification service:

1. Create `apps/mobile/src/services/api/notification.api.ts`:
   ```typescript
   // API methods using unifiedApiClient
   - getNotifications(params?: { limit?, offset?, unreadOnly? })
   - markAsRead(notificationId: string)
   - markAllAsRead()
   - deleteNotification(notificationId: string)
   - getNotificationSettings()
   - updateNotificationSettings(settings: PushNotificationSettings)
   - registerDeviceToken(token: string, platform: 'ios' | 'android')
   - deregisterDeviceToken(token: string)
   ```

2. Export from `apps/mobile/src/services/api/index.ts`

3. Create `apps/mobile/src/services/push-notification.service.ts`:
   - A service class (not React component) that:
     - On app startup, requests notification permission via react-native's PushNotificationIOS / Firebase messaging
     - Gets the FCM/APNs token and registers it with the backend
     - Listens for incoming push notifications (foreground and background)
     - When notification received: shows local notification, updates badge count
     - Handles notification tap: navigates to the relevant screen based on notification data
     - On token refresh: re-registers with backend
     - Uses a simple approach compatible with React Native (not Expo push):
       - For Android: `@react-native-firebase/messaging` (already in project dependencies or need to add)
       - For iOS: same package handles APNs via Firebase
     - Gracefully handles when Firebase is not configured (dev environment)
</action>
<acceptance_criteria>
- notification.api.ts with all 8 API methods using unifiedApiClient
- Push notification service with permission request, token registration, and notification handling
- Graceful degradation when push not configured
- Exported from API index
</acceptance_criteria>
</task>

Create `apps/mobile/src/stores/notificationStore.ts`:
```typescript
interface NotificationState {
  notifications: AppNotification[];
  unreadCount: number;
  loading: boolean;
  hasMore: boolean;
  settings: PushNotificationSettings | null;

  // Actions
  fetchNotifications: (reset?: boolean) => Promise<void>;
  markAsRead: (id: string) => Promise<void>;
  markAllAsRead: () => Promise<void>;
  deleteNotification: (id: string) => Promise<void>;
  fetchSettings: () => Promise<void>;
  updateSettings: (settings: Partial<PushNotificationSettings>) => Promise<void>;
  decrementUnread: () => void;
  reset: () => void;
}

interface PushNotificationSettings {
  order: boolean;
  recommendation: boolean;
  community: boolean;
  system: boolean;
  quietHoursEnabled: boolean;
  quietHoursStart: string;
  quietHoursEnd: string;
}

interface AppNotification {
  id: string;
  type: string;
  category: 'order' | 'recommendation' | 'community' | 'system';
  title: string;
  content: string;
  data?: Record<string, unknown>;
  targetType?: string;
  targetId?: string;
  isRead: boolean;
  readAt?: string;
  createdAt: string;
}
```

- Fetch notifications from backend API with pagination (limit=20, offset-based)
- Optimistic updates for mark-as-read (update local state immediately, fire API in background)
- Export from stores/index.ts
