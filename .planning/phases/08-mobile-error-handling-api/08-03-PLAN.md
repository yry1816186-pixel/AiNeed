- apps/mobile/src/features/commerce/stores/cart.store.ts
- apps/mobile/src/services/api/ (检查可用的 API 服务)
</read_first>

<action>
根据 Task 1 的 API 可用性检查结果，为每个空壳 Store 添加 fetch 方法。

**统一模式 — API 已实现时：**
```typescript
fetchXxx: async () => {
  set({ isLoading: true, error: null });
  try {
    const response = await xxxApi.getXxx();
    set({ items: response.data, isLoading: false });
  } catch (error) {
    set({ error: '获取XXX失败，请稍后重试', isLoading: false });
  }
}
```

**统一模式 — API 未实现时：**
```typescript
fetchXxx: async () => {
  set({ isLoading: true, error: null });
  try {
    // TODO: 连接后端 API 后替换
    set({ error: '功能开发中，敬请期待', isLoading: false });
  } catch (error) {
    set({ error: '操作失败', isLoading: false });
  }
}
```

需要处理的 Store 方法：
1. useWardrobeStore: fetchUserClothing, fetchOutfits
2. useClothingStore: fetchClothing, fetchFeatured, fetchTrending, fetchNewArrivals
3. useRecommendationStore: fetchRecommendations, fetchSimilarItems
4. useAnalysisStore: fetchClothingAnalysis, fetchBodyAnalysis
5. useCartStore: syncCart, fetchCart

每个方法根据 Task 1 的 API 可用性结果选择对应模式实现。
</action>

<acceptance_criteria>
- `grep "fetchUserClothing" apps/mobile/src/features/wardrobe/stores/wardrobeStore.ts` 返回至少 1 行
- `grep "fetchCart\|syncCart" apps/mobile/src/features/commerce/stores/cart.store.ts` 返回至少 1 行
- `grep "fetchRecommendations" apps/mobile/src/features/home/stores/recommendation.store.ts` 返回至少 1 行
- 每个 fetch 方法包含 try/catch 和 error 状态管理
</acceptance_criteria>
