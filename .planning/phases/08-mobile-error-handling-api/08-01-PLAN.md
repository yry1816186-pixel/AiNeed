# Plan 01: 关键 Store 添加 error 状态字段

## Objective

为 6 个有异步操作但缺少 error 状态的 Store 添加 `error: string | null` 字段、`setError(message: string)` 和 `clearError()` 方法，确保所有异步操作失败时用户可感知。

## Tasks

### Task 1: useOrderStore 添加 error 状态

<read_first>
- apps/mobile/src/features/commerce/stores/orderStore.ts
- apps/mobile/src/features/auth/stores/authStore.ts (参考 error 状态模式)
</read_first>

<action>
在 `apps/mobile/src/features/commerce/stores/orderStore.ts` 中：

1. 在 state interface 中添加 `error: string | null` 字段，初始值 `null`
2. 添加 `setError: (message: string) => void` 方法，实现 `set({ error: message })`
3. 添加 `clearError: () => void` 方法，实现 `set({ error: null })`
4. 在 `fetchOrdersByTab` 的 catch 块中，替换 `console.error` 为：
   ```typescript
   set({ error: '获取订单失败，请稍后重试', isLoading: false });
   ```
5. 在 `confirmReceipt` 的 catch 块中，替换 `console.error` 为：
   ```typescript
   set({ error: '确认收货失败，请稍后重试', isLoading: false });
   ```
6. 在 `softDeleteOrder` 的 catch 块中，替换 `console.error` 为：
   ```typescript
   set({ error: '删除订单失败，请稍后重试', isLoading: false });
   ```
7. 在每个方法的 try 块开头添加 `set({ isLoading: true, error: null });` 清除旧错误
</action>

<acceptance_criteria>
- `orderStore.ts` 包含 `error: string | null` 状态字段
- `orderStore.ts` 包含 `setError` 和 `clearError` 方法
- `fetchOrdersByTab` catch 块设置中文 error 消息
- `confirmReceipt` catch 块设置中文 error 消息
- `softDeleteOrder` catch 块设置中文 error 消息
- 所有 try 块开头清除旧 error
- `grep -c "error:" apps/mobile/src/features/commerce/stores/orderStore.ts` >= 6
</acceptance_criteria>

## must_haves

- 用户在订单/优惠券/通知操作失败时能看到错误提示

- 错误消息使用中文
- 不破坏现有 Store 功能
