## Task 4: 深色模式阴影与 elevation 修复

<read_first>
- apps/mobile/src/design-system/theme/tokens/shadows.ts
- apps/mobile/src/design-system/ui/PaperThemeProvider.tsx
- apps/mobile/src/shared/contexts/ThemeContext.tsx
</read_first>

<action>
1. 审计所有使用 `elevation:` 数字值的文件（约92处），对关键组件添加暗色模式阴影支持:
   - 在深色模式下，为使用 elevation 的组件添加:
     ```typescript
     shadowColor: colors.textPrimary,
     shadowOffset: { width: 0, height: 2 },
     shadowOpacity: isDark ? 0.3 : 0.1,
     shadowRadius: 4,
     ```
   - 优先处理用户可见的浮动元素: FAB、浮动卡片、弹窗

2. 确保 PaperThemeProvider 的 elevation 配置在深色模式下有层级差异:
   - 当前所有 level1-5 都使用 `c.backgrounds.elevated`，缺乏层级感
   - 修改为:
     ```typescript
     // Dark theme elevation
     level0: "transparent",
     level1: c.backgrounds.elevated,     // #1F1B18
     level2: adjustElevation(c.backgrounds.elevated, 0.02),  // slightly lighter
     level3: adjustElevation(c.backgrounds.elevated, 0.04),
     level4: adjustElevation(c.backgrounds.elevated, 0.06),
     level5: adjustElevation(c.backgrounds.elevated, 0.08),
     ```
   - 或使用 darkTokens 中预定义的 elevation 层级颜色

3. 如果 darkTokens 中没有预定义 elevation 层级颜色，在 design-tokens.ts 中添加:
   ```typescript
   backgrounds: {
     ...,
     elevated: "#1F1B18",
     elevated2: "#252019",
     elevated3: "#2A251E",
     elevated4: "#302A23",
     elevated5: "#362F28",
   }
   ```
</action>

<acceptance_criteria>
- PaperThemeProvider 深色模式 elevation level1-5 使用不同颜色值
- `grep "level1.*level2.*level3" apps/mobile/src/design-system/ui/PaperThemeProvider.tsx` 显示不同值
- 关键浮动组件（FAB、浮动卡片）在深色模式下有可见阴影
- TypeScript 编译通过
</acceptance_criteria>
