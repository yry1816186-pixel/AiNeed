## Task 1: 开启 dark_mode feature flag 并在 ThemeContext 中消费

<read_first>
- apps/mobile/src/constants/feature-flags.ts
- apps/mobile/src/shared/contexts/ThemeContext.tsx
- apps/mobile/src/shared/contexts/FeatureFlagContext.tsx
</read_first>

<action>
1. 在 `apps/mobile/src/constants/feature-flags.ts` 中：
   - 将第21行 `dark_mode: false` 改为 `dark_mode: true`

2. 在 `apps/mobile/src/shared/contexts/ThemeContext.tsx` 中：
   - 添加 import: `import { useFeatureFlags } from "./FeatureFlagContext";`
   - 在 ThemeProvider 组件内部添加: `const { isEnabled } = useFeatureFlags();`
   - 在 isDark 计算逻辑中增加 feature flag 检查:

3. 确保 ThemeProvider 能正确访问 FeatureFlagProvider。检查 App.tsx 中的 Provider 嵌套顺序，如果 FeatureFlagProvider 在 ThemeProvider 外部则无需调整；如果在内部，需要将 FeatureFlagProvider 移到 ThemeProvider 外部或使用独立读取方式。
</action>

- `ThemeContext.tsx` 中 isDark 计算包含 `dark_mode` 标志检查
- 当 dark_mode flag 关闭时，isDark 始终为 false
</acceptance_criteria>
