# Plan 02: 替换紫色/品牌冲突色为 Terracotta 品牌色

## Objective

将所有紫色/粉色硬编码颜色（#a855f7, #ec4899, #6C5CE7, #F3F1FF 等）替换为 Terracotta 品牌色系令牌引用。这是最显眼的品牌不一致问题，按 D-12 优先修复。

## Threat Model

- 无安全威胁：纯 UI 颜色替换
- 风险：颜色替换可能影响视觉一致性，需逐文件验证编译通过

## Tasks

### Task 02-01: ProfileSetupFlow 紫色/粉色渐变替换

<objective>
将 ProfileSetupFlow 中的紫色/粉色渐变替换为 Terracotta 品牌色渐变。
</objective>

<read_first>
- apps/mobile/src/shared/components/flows/ProfileSetupFlow.tsx
- apps/mobile/src/components/flows/ProfileSetupFlow.tsx
- apps/mobile/src/design-system/theme/tokens/design-tokens.ts (DesignTokens.colors.brand, DesignTokens.gradients)
- apps/mobile/src/design-system/theme/index.ts (Colors, gradients imports)
</read_first>

<action>
对两个 ProfileSetupFlow 文件执行相同替换：

1. 添加导入（如果尚未存在）：
   ```typescript
   import { DesignTokens } from '../../design-system/theme/tokens/design-tokens';
   ```
   或使用已有的 theme/Colors 导入

2. 替换所有颜色值：
   - `#a855f7` → `DesignTokens.colors.brand.terracotta` (即 '#C67B5C')
   - `#ec4899` → `DesignTokens.colors.brand.camel` (即 '#B5A08C')
   - `rgba(168, 85, 247, 0.x)` → `rgba(198, 123, 92, 0.x)` (Terracotta with same opacity)
   - `rgba(236, 72, 153, 0.x)` → `rgba(181, 160, 140, 0.x)` (Camel with same opacity)
   - 紫色渐变 `['#a855f7', '#ec4899']` → `DesignTokens.gradients.brand` (即 ['#C67B5C', '#B5A08C'])
   - 紫色背景 `#F3F1FF` → `DesignTokens.colors.backgrounds.tertiary` (即 '#F5F5F3')
   - 紫色文字 `#7C3AED` → `DesignTokens.colors.brand.terracottaDark` (即 '#A86548')

3. 如果文件使用 StyleSheet.create 中的硬编码颜色，替换为 DesignTokens 引用
4. 如果文件使用内联样式中的硬编码颜色，同样替换
</action>

<acceptance_criteria>
- 两个 ProfileSetupFlow 文件均不包含 `#a855f7`
- 两个 ProfileSetupFlow 文件均不包含 `#ec4899`
- 两个 ProfileSetupFlow 文件均不包含 `#7C3AED`
- 两个 ProfileSetupFlow 文件均包含 `DesignTokens.colors.brand.terracotta` 或等效令牌引用

</acceptance_criteria>
