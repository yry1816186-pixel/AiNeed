### Task 03-02: 屏幕文件通用硬编码颜色替换

<objective>
将屏幕文件中所有剩余的硬编码十六进制颜色替换为 DesignTokens 令牌引用。按语义上下文分类替换。
</objective>

<read_first>
- apps/mobile/src/design-system/theme/tokens/design-tokens.ts
- apps/mobile/src/design-system/theme/index.ts
- 所有 screens/ 和 features/*/screens/ 下的 .tsx 文件
</read_first>

<action>
对每个屏幕文件：

1. 扫描所有硬编码颜色值（#xxx 或 #xxxxxx 格式）
2. 按语义上下文分类替换：
   - 文字颜色 → `DesignTokens.colors.text.*`
     - 深色文字 (#1A1A2E, #000000, #333, #222) → `DesignTokens.colors.text.primary`
     - 次要文字 (#666, #888, #999, #6B7280) → `DesignTokens.colors.text.secondary`
     - 提示文字 (#aaa, #bbb, #ccc) → `DesignTokens.colors.text.tertiary`
     - 白色文字 (#fff, #ffffff) → `DesignTokens.colors.text.inverse`
   - 背景颜色 → `DesignTokens.colors.backgrounds.*`
     - 白色背景 (#fff, #ffffff) → `DesignTokens.colors.backgrounds.primary`
     - 浅灰背景 (#f5f5f5, #fafafa, #f8f9fa) → `DesignTokens.colors.backgrounds.secondary` 或 `.tertiary`
     - 卡片/弹窗背景 → `DesignTokens.colors.backgrounds.elevated`
   - 边框颜色 → `DesignTokens.colors.borders.*`
     - 浅边框 (#eee, #e5e5e5) → `DesignTokens.colors.borders.light`
     - 默认边框 (#ddd, #ccc) → `DesignTokens.colors.borders.default`
   - 品牌色 → `DesignTokens.colors.brand.*`
     - 主品牌色 → `DesignTokens.colors.brand.terracotta`
   - 语义色 → `DesignTokens.colors.semantic.*`

3. 添加必要的 DesignTokens 导入
4. 对于 rgba() 颜色，尽量使用 DesignTokens 中的对应值；如无对应，保留 rgba 但添加注释说明原因

每替换一个文件后运行 `cd apps/mobile && npx tsc --noEmit` 验证编译通过。
</action>

<acceptance_criteria>
- `grep -rn "#[0-9A-Fa-f]\{3,6\}" apps/mobile/src/screens/ apps/mobile/src/features/ --include="*.tsx" | grep -v "design-system/" | grep -v "node_modules/" | wc -l` 返回 0（或仅剩 token 定义文件中的颜色）
- `cd apps/mobile && npx tsc --noEmit` exits with code 0
</acceptance_criteria>
