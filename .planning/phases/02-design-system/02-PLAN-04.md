# Plan 04: 间距 Token 迁移 Codemod

## Objective

将所有硬编码间距值（~1,174 处 padding/margin/gap/width/height）迁移至 DesignTokens.spacing.* 语义化 Token。

## Tasks

### Task 04-01: 创建语义化间距映射表和补充 Token

<objective>
创建硬编码间距值到语义化 Token 的映射表。当前 SpacingScale 是数字型的（Spacing[0]=0, Spacing[1]=4, Spacing[2]=8...），需要添加语义化别名。
</objective>

<read_first>
- apps/mobile/src/theme/tokens/spacing.ts
- apps/mobile/src/theme/tokens/design-tokens.ts
- apps/mobile/scripts/audit-hardcoded-spacing.ts
</read_first>

<action>
1. 运行审计脚本：`cd apps/mobile && npx ts-node scripts/audit-hardcoded-spacing.ts > scripts/spacing-audit-report.json`

2. 在 `apps/mobile/src/theme/tokens/spacing.ts` 中添加语义化间距别名：

```typescript
export const SemanticSpacing = {
  padding: {
    screen: SpacingScale[4],      // 16 - 屏幕边距
    card: SpacingScale[3],        // 12 - 卡片内边距
    cardLarge: SpacingScale[4],   // 16 - 大卡片内边距
    section: SpacingScale[5],     // 20 - 区块内边距
    input: SpacingScale[3],       // 12 - 输入框内边距
    button: SpacingScale[3],      // 12 - 按钮内边距
    chip: SpacingScale[2],        // 8 - 标签内边距
    listItem: SpacingScale[4],    // 16 - 列表项内边距
    dialog: SpacingScale[6],      // 24 - 对话框内边距
  },
  gap: {
    xs: SpacingScale[1],          // 4
    sm: SpacingScale[2],          // 8
    md: SpacingScale[3],          // 12
    lg: SpacingScale[4],          // 16
    xl: SpacingScale[5],          // 20
    xxl: SpacingScale[6],         // 24
    card: SpacingScale[3],        // 12 - 卡片间距
    section: SpacingScale[6],     // 24 - 区块间距
    list: SpacingScale[2],        // 8 - 列表项间距
  },
  margin: {
    sm: SpacingScale[1],          // 4
    md: SpacingScale[2],          // 8
    lg: SpacingScale[3],          // 12
    xl: SpacingScale[4],          // 16
    section: SpacingScale[6],     // 24 - 区块外边距

```

### Task 04-02: 创建间距迁移 Codemod 并执行

<objective>
编写 ts-morph codemod 脚本，将硬编码间距值替换为 SemanticSpacing 引用，然后执行迁移。
</objective>

<read_first>
- apps/mobile/scripts/semantic-spacing-map.json
- apps/mobile/src/theme/tokens/spacing.ts
- apps/mobile/src/theme/tokens/design-tokens.ts
</read_first>

<action>
1. 创建 `apps/mobile/scripts/codemod-spacing.ts`：

```typescript
// 使用 ts-morph 解析 TSX/TS 文件
// 逻辑:
// 1. 加载 semantic-spacing-map.json
// 2. 遍历 src/ 下所有 .tsx/.ts 文件（排除 node_modules, theme/tokens/）
// 3. 在 StyleSheet.create() 和内联样式中查找间距属性
// 4. 替换规则:
//    - padding/paddingTop/paddingBottom/paddingLeft/paddingRight/paddingHorizontal/paddingVertical
//      → SemanticSpacing.padding.xxx
//    - margin/marginTop/marginBottom/marginLeft/marginRight/marginHorizontal/marginVertical
//      → SemanticSpacing.margin.xxx
//    - gap → SemanticSpacing.gap.xxx
//    - width/height: 仅替换常见间距值（如 1 分隔线），布局值保留
// 5. 排除值: 0, 0.5, 1（分隔线除外）, 百分比值, 'auto'
// 6. 添加 import { SemanticSpacing } from '...' 如果不存在
// 7. 保留原始值作为注释: /* was: 16 */
// 8. 支持 --dry-run 和 --file <path> 模式
//
// 特殊处理:
// - 已使用 Spacing[4] / Spacing.md 的文件: 替换为 SemanticSpacing 等价物
// - Spacing[4] 在 padding 上下文 → SemanticSpacing.padding.screen
// - Spacing[4] 在 gap 上下文 → SemanticSpacing.gap.lg
```

2. 在 `apps/mobile/package.json` 中添加脚本：

```json
{
  "scripts": {
    "codemod:spacing": "npx ts-node scripts/codemod-spacing.ts",
    "codemod:spacing:dry-run": "npx ts-node scripts/codemod-spacing.ts --dry-run"
  }
}
```

3. 执行迁移：
   - 先 dry-run：`cd apps/mobile && npx ts-node scripts/codemod-spacing.ts --dry-run`
   - 审查报告
   - 执行：`cd apps/mobile && npx ts-node scripts/codemod-spacing.ts`
   - 类型检查：`cd apps/mobile && npx tsc --noEmit`

4. 手动处理无法自动映射的值（布局相关的 width/height 等）
</action>

<acceptance_criteria>
- `apps/mobile/scripts/codemod-spacing.ts` 文件存在
- `apps/mobile/package.json` 包含 `codemod:spacing` 脚本
- `cd apps/mobile && npx tsc --noEmit` 无新增类型错误
- `cd apps/mobile && npx ts-node scripts/audit-hardcoded-spacing.ts` 报告用户可见组件中硬编码间距 < 50 处（从 ~1,174 降低）
- 原始值保留为注释 `/* was: 16 */`
</acceptance_criteria>

<verification>
运行审计脚本和类型检查，确认迁移进度。
</verification>

### Task 04-03: 手动处理剩余间距值

<objective>
处理 codemod 无法自动映射的间距值，确保用户可见组件中 0 处硬编码间距。
</objective>

<read_first>
- apps/mobile/scripts/spacing-audit-report.json
- apps/mobile/src/theme/tokens/spacing.ts
</read_first>

<action>
1. 读取 spacing-migration-report.json 中的 `skipped` 列表

2. 对每个跳过的值进行分类处理：
   - **布局值**（width/height 用于布局）：保留为硬编码，添加 `// layout-value` 注释
   - **非标准间距值**（如 3, 5, 7）：添加到 SemanticSpacing 或使用 SpacingScale 直接引用
   - **百分比/比例值**：保留，这些是布局值而非间距值
   - **0 值**：保留为 `0`（不需要 token 化）

3. 在 SemanticSpacing 中补充缺失的 token

4. 手动替换剩余硬编码值
</action>

<acceptance_criteria>
- `cd apps/mobile && npx ts-node scripts/audit-hardcoded-spacing.ts` 报告用户可见组件中硬编码间距 = 0 处（排除布局值和 0 值）
- 所有保留的布局值标记为 `// layout-value`
</acceptance_criteria>

<verification>
运行审计脚本，确认硬编码间距为 0。
</verification>
