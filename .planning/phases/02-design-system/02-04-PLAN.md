# Plan 04: 组件文件硬编码颜色迁移 + ClothingCard 令牌化

## Objective

将所有组件文件中的硬编码十六进制颜色替换为 DesignTokens 令牌引用。重点处理 ClothingCard 自定义颜色映射。

## Threat Model

- 无安全威胁：纯 UI 颜色替换
- 风险：ClothingCard 颜色映射涉及业务逻辑，需保持映射关系正确

## Tasks

### Task 04-01: ClothingCard 自定义颜色映射令牌化

<objective>
将 ClothingCard 中的自定义颜色映射改为使用 DesignTokens.colors 令牌。
</objective>

<read_first>
- apps/mobile/src/features/wardrobe/components/ClothingCard.tsx
- apps/mobile/src/components/clothing/ClothingCard.tsx
- apps/mobile/src/design-system/theme/tokens/design-tokens.ts
- apps/mobile/src/design-system/theme/tokens/colors.ts
</read_first>

<action>
对两个 ClothingCard 文件执行相同替换：

1. 找到自定义颜色映射对象（通常是 category → color 的映射）
2. 将硬编码颜色值替换为 DesignTokens 令牌引用：
   - 红色系 → `DesignTokens.colors.brand.terracotta` 或 `DesignTokens.colors.semantic.error`
   - 蓝色系 → `DesignTokens.colors.brand.slate` 或 `DesignTokens.colors.semantic.info`
   - 绿色系 → `DesignTokens.colors.brand.sage` 或 `DesignTokens.colors.semantic.success`
   - 黄色系 → `DesignTokens.colors.semantic.warning`
   - 紫色系 → `DesignTokens.colors.brand.terracotta` (品牌色替代)
   - 粉色系 → `DesignTokens.colors.brand.terracottaLight`
   - 橙色系 → `DesignTokens.colors.brand.terracotta`
   - 灰色系 → `DesignTokens.colors.neutral[xxx]`

3. 如果映射是动态的（如 `colorMap[category]`），保持映射结构但替换值为令牌
4. 添加 DesignTokens 导入
</action>

<acceptance_criteria>
- ClothingCard 文件中无硬编码 hex 颜色（除 token 定义外）
- ClothingCard 颜色映射使用 DesignTokens.colors.* 引用
- `cd apps/mobile && npx tsc --noEmit` exits with code 0
</acceptance_criteria>

### Task 04-03: 组件文件通用硬编码颜色替换

<objective>
将组件文件中所有剩余的硬编码十六进制颜色替换为 DesignTokens 令牌引用。
</objective>

<read_first>
- apps/mobile/src/design-system/theme/tokens/design-tokens.ts
- apps/mobile/src/design-system/theme/index.ts
- 所有 components/, features/*/components/, shared/components/ 下的 .tsx 文件
</read_first>

<action>
对每个组件文件执行与 Plan 03 Task 03-02 相同的语义分类替换策略：

1. 扫描所有硬编码颜色值
2. 按语义上下文分类替换（text/backgrounds/borders/brand/semantic）
3. 添加 DesignTokens 导入
4. 每替换一个文件后验证编译通过

注意：
- 组件文件中的颜色通常与交互状态相关（active, pressed, disabled），需选择正确的语义令牌
- 渐变色使用 DesignTokens.gradients.* 令牌
- 透明色使用 DesignTokens.colors.backgrounds.overlay 或保留 rgba
</action>

<acceptance_criteria>
- `grep -rn "#[0-9A-Fa-f]\{3,6\}" apps/mobile/src/components/ apps/mobile/src/features/ apps/mobile/src/shared/ --include="*.tsx" | grep -v "design-system/" | grep -v "node_modules/" | wc -l` 返回 0（或仅剩无法替换的特殊值）
- `cd apps/mobile && npx tsc --noEmit` exits with code 0
</acceptance_criteria>
