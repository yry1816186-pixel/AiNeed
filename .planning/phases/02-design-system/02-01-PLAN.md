# Plan 01: 统一令牌定义冲突 + 废弃 PlayfairDisplay

## Objective

解决 DesignTokens 与 typography.ts/spacing.ts 的同名语义值冲突，以 DesignTokens 为准统一所有令牌定义。废弃无人使用的 PlayfairDisplay 字体。修复 theme.colors.* API 兼容性。

## Threat Model

- 无安全威胁：纯 UI 令牌值修改，不涉及 PII、网络请求或认证逻辑
- 低风险：修改的是静态常量定义，影响范围可通过 TypeScript 编译验证

## Tasks

### Task 01-01: 统一 typography.ts FontSizes 与 DesignTokens.typography.sizes 冲突

<objective>
将 typography.ts 中 FontSizes 的冲突值修改为与 DesignTokens.typography.sizes 一致。
</objective>

<read_first>
- apps/mobile/src/design-system/theme/tokens/design-tokens.ts (DesignTokens.typography.sizes: xs=11, sm=12, base=14, md=16)
- apps/mobile/src/design-system/theme/tokens/typography.ts (FontSizes: xs=12, sm=14, base=16)
</read_first>

<action>
1. 在 `apps/mobile/src/design-system/theme/tokens/typography.ts` 中修改 `FontSizes` 对象：
   - 将 `xs: 12` 改为 `xs: 11`
   - 将 `sm: 14` 改为 `sm: 12`
   - 将 `base: 16` 改为 `base: 14`
   - 保留所有其他值不变（lg: 18, xl: 20, 2xl: 24, 3xl: 30, 4xl: 36, 5xl: 48, 6xl: 60, 7xl: 72, 8xl: 96, hero: 64）
   - 添加 `md: 16` 键（DesignTokens 有 md:16 但 FontSizes 缺少）
2. 在文件顶部添加注释说明值来源：
   `// FontSizes aligned with DesignTokens.typography.sizes (DesignTokens is source of truth)`
</action>

<acceptance_criteria>
- `typography.ts` contains `xs: 11` (not `xs: 12`)
- `typography.ts` contains `sm: 12` (not `sm: 14`)
- `typography.ts` contains `base: 14` (not `base: 16`)
- `typography.ts` contains `md: 16`
- `cd apps/mobile && npx tsc --noEmit` exits with code 0
</acceptance_criteria>

### Task 01-04: 修复 theme.colors.* API — 确保扁平属性返回字符串

<objective>
确保 theme.colors 的扁平属性（text, primary, surface 等）返回字符串而非嵌套对象。当前 flatColors.text 返回的是 DesignTokens.colors.text 对象而非字符串。
</objective>

<read_first>
- apps/mobile/src/design-system/theme/index.ts (buildFlatThemeColors function, theme export)
- apps/mobile/src/design-system/theme/FlatColors.ts (FlatColors interface)
</read_first>

<action>
1. 在 `apps/mobile/src/design-system/theme/index.ts` 的 `buildFlatThemeColors` 函数中，将嵌套对象属性替换为字符串属性：
   - 将 `text: base.text` 改为 `text: base.text.primary`（即 `'#1A1A18'`）
   - 将 `brand: base.brand` 改为 `brand: base.brand.terracotta`（即 `'#C67B5C'`）
   - 将 `neutral: base.neutral` 改为 `neutral: base.neutral[600]`（即 `'#52524D'`）
   - 将 `semantic: base.semantic` 改为 `semantic: base.semantic.info`（即 `'#7B8FA2'`）
   - 将 `backgrounds: base.backgrounds` 改为 `backgrounds: base.backgrounds.primary`（即 `'#FFFFFF'`）
   - 将 `borders: base.borders` 改为 `borders: base.borders.default`（即 `'rgba(0, 0, 0, 0.1)'`）
   - 将 `colorSeasons: base.colorSeasons` 改为 `colorSeasons: base.colorSeasons.spring.bg`（即 `'#FFF5F0'`）
2. 在 `apps/mobile/src/design-system/theme/FlatColors.ts` 中，将嵌套类型改为 string：
   - 将 `brand: TokenSet["colors"]["brand"]` 改为 `brand: string`
   - 将 `neutral: TokenSet["colors"]["neutral"]` 改为 `neutral: string`
   - 将 `semantic: TokenSet["colors"]["semantic"]` 改为 `semantic: string`
   - 将 `backgrounds: TokenSet["colors"]["backgrounds"]` 改为 `backgrounds: string`
   - 将 `text: TokenSet["colors"]["text"]` 改为 `text: string`
   - 将 `borders: TokenSet["colors"]["borders"]` 改为 `borders: string`
   - 将 `colorSeasons: TokenSet["colors"]["colorSeasons"]` 改为 `colorSeasons: string`
3. 在 `index.ts` 中添加新的嵌套访问属性（保留向后兼容）：
   ```typescript

     brand: DesignTokens.colors.brand,
     neutral: DesignTokens.colors.neutral,

     backgrounds: DesignTokens.colors.backgrounds,
     text: DesignTokens.colors.text,
     borders: DesignTokens.colors.borders,
     colorSeasons: DesignTokens.colors.colorSeasons,

   ```
   并在 theme 对象中添加 `themeColors` 导出
</action>

<acceptance_criteria>

- `typeof theme.colors.neutral === 'string'` (not object)
