# Plan 05: 暗色模式迁移 — 颜色从 StyleSheet.create 到 useTheme()

## Objective

将颜色从静态 `StyleSheet.create` 迁移到响应式 `useTheme()` hook，使暗色模式正常工作。这是最复杂的步骤，因为需要将颜色从静态样式对象中提取出来，改为运行时动态获取。

## Tasks

### Task 05-01: 创建 useThemeStyles Hook

<objective>
创建 `useThemeStyles` hook，允许组件在 StyleSheet.create 模式下使用响应式主题颜色。这是暗色模式迁移的关键工具。
</objective>

<read_first>
- apps/mobile/src/contexts/ThemeContext.tsx
- apps/mobile/src/hooks/useThemeColor.ts
- apps/mobile/src/theme/tokens/design-tokens.ts
</read_first>

<action>
1. 创建 `apps/mobile/src/hooks/useThemeStyles.ts`：

```typescript
import { useMemo } from 'react';
import { StyleSheet, TextStyle, ViewStyle, ImageStyle } from 'react-native';
import { useTheme } from '../contexts/ThemeContext';

type NamedStyles<T> = { [P in keyof T]: ViewStyle | TextStyle | ImageStyle };

export function useThemeStyles<T extends NamedStyles<T>>(
  createStyles: (colors: typeof import('../theme/tokens/design-tokens').DesignTokens.colors) => T
): { styles: T; colors: ReturnType<typeof useTheme>['colors'] } {
  const { colors } = useTheme();

  const styles = useMemo(() => StyleSheet.create(createStyles(colors)), [colors]);

  return { styles, colors };
}
```

2. 使用模式对比：

**迁移前（静态，暗色模式不工作）：**
```typescript
import { DesignTokens } from '../theme/tokens/design-tokens';

const styles = StyleSheet.create({
  container: {
    backgroundColor: DesignTokens.colors.backgrounds.primary,
  },
  text: {
    color: DesignTokens.colors.text.primary,
  },
});
```

**迁移后（响应式，暗色模式工作）：**
```typescript
import { useThemeStyles } from '../hooks/useThemeStyles';

function MyComponent() {
  const { styles, colors } = useThemeStyles((colors) => StyleSheet.create({
    container: {
      backgroundColor: colors.backgrounds.primary,
    },
    text: {
      color: colors.text.primary,
    },
  }));

  return <View style={styles.container}><Text style={styles.text}>Hello</Text></View>;
}
```

3. 在 `apps/mobile/src/hooks/index.ts` 中导出 `useThemeStyles`
</action>

- 导出 `useThemeStyles` 函数
- `useThemeStyles` 接受一个工厂函数参数，返回 `{ styles, colors }`
- `cd apps/mobile && npx tsc --noEmit` 无新增类型错误
</acceptance_criteria>
