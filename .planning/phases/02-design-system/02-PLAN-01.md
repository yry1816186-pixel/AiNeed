### Task 01-02: 创建 ESLint 规则阻止新增硬编码样式值

<objective>
在移动端 ESLint 配置中添加规则，阻止新增硬编码颜色、fontSize 和间距值，确保迁移后不会回退。
</objective>

<read_first>
- apps/mobile/.eslintrc.json
- apps/mobile/package.json
</read_first>

<action>
1. 安装 `eslint-plugin-react-native-design-system`（如无合适插件，则创建自定义规则文件 `apps/mobile/eslint-rules/no-hardcoded-styles.js`）：

```javascript
// apps/mobile/eslint-rules/no-hardcoded-styles.js
module.exports = {
  meta: { type: 'suggestion', severity: 'error' },
  create(context) {
    return {
      Property(node) {
        if (!node.key) return;
        const key = node.key.name;
        // 颜色属性检查
        const colorProps = ['color', 'backgroundColor', 'borderColor', 'borderBottomColor', 'borderTopColor', 'borderLeftColor', 'borderRightColor', 'tintColor', 'placeholderTextColor', 'underlineColorAndroid'];
        if (colorProps.includes(key)) {
          const value = node.value;
          if (value.type === 'Literal' && typeof value.value === 'string') {
            const v = value.value;
            if (/^#([0-9a-fA-F]{3,8})$/.test(v) || /^rgba?\(/.test(v) || /^rgb\(/.test(v)) {
              context.report({ node, message: `硬编码颜色值 "${v}" 不允许，请使用 DesignTokens.colors.* 或 useTheme()` });
            }
          }
        }
        // fontSize 检查
        if (key === 'fontSize') {
          const value = node.value;
          if (value.type === 'Literal' && typeof value.value === 'number') {
            context.report({ node, message: `硬编码 fontSize ${value.value} 不允许，请使用 DesignTokens.typography.*` });
          }
        }
      }
    };
  }
};
```

2. 在 `apps/mobile/.eslintrc.json` 中添加：

```json
{
  "plugins": ["../../eslint-rules"],
  "rules": {
    "no-hardcoded-styles": "warn"
  }
}
```

3. 注意：初始设置为 `warn` 而非 `error`，待 Phase 2 完成后改为 `error`。
</action>

<acceptance_criteria>
- `apps/mobile/eslint-rules/no-hardcoded-styles.js` 文件存在
- `apps/mobile/.eslintrc.json` 包含 `"no-hardcoded-styles": "warn"` 规则
- `cd apps/mobile && npx eslint src/screens/HomeScreen.tsx --no-eslintrc --rulesdir eslint-rules 2>&1` 能检测到硬编码颜色
</acceptance_criteria>
