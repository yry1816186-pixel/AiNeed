# Plan 05: 硬编码 fontSize 和间距迁移

## Objective

将所有硬编码 fontSize 和间距值替换为 DesignTokens 令牌引用。覆盖所有 .tsx 和 .ts 文件。

## Threat Model

- 无安全威胁：纯 UI 样式值替换
- 风险：fontSize 替换可能影响布局，需验证编译通过

## Tasks

### Task 05-01: 硬编码 fontSize 替换

<objective>
将所有硬编码 fontSize 数字替换为 DesignTokens.typography.sizes 令牌引用。
</objective>

<read_first>
- apps/mobile/src/design-system/theme/tokens/design-tokens.ts (DesignTokens.typography.sizes: xs=11, sm=12, base=14, md=16, lg=18, xl=20, 2xl=24, 3xl=30, 4xl=36, 5xl=48, 6xl=60)
- apps/mobile/src/design-system/theme/index.ts (Typography.sizes, Typography.styles)
- 所有包含硬编码 fontSize 的 .tsx 文件
</read_first>

<action>
对所有文件中的硬编码 fontSize 执行替换：

1. 直接映射（值与 DesignTokens 完全匹配）：
   - `fontSize: 11` → `fontSize: DesignTokens.typography.sizes.xs`
   - `fontSize: 12` → `fontSize: DesignTokens.typography.sizes.sm`
   - `fontSize: 14` → `fontSize: DesignTokens.typography.sizes.base`
   - `fontSize: 16` → `fontSize: DesignTokens.typography.sizes.md`
   - `fontSize: 18` → `fontSize: DesignTokens.typography.sizes.lg`
   - `fontSize: 20` → `fontSize: DesignTokens.typography.sizes.xl`
   - `fontSize: 24` → `fontSize: DesignTokens.typography.sizes['2xl']`
   - `fontSize: 30` → `fontSize: DesignTokens.typography.sizes['3xl']`
   - `fontSize: 36` → `fontSize: DesignTokens.typography.sizes['4xl']`
   - `fontSize: 48` → `fontSize: DesignTokens.typography.sizes['5xl']`
   - `fontSize: 60` → `fontSize: DesignTokens.typography.sizes['6xl']`

2. 近似映射（值不在 DesignTokens 中，映射到最近值）：
   - `fontSize: 10` → `fontSize: DesignTokens.typography.sizes.xs` (11, 差1px)
   - `fontSize: 13` → `fontSize: DesignTokens.typography.sizes.sm` (12, 差1px)
   - `fontSize: 15` → `fontSize: DesignTokens.typography.sizes.base` (14, 差1px)
   - `fontSize: 17` → `fontSize: DesignTokens.typography.sizes.md` (16, 差1px)
   - `fontSize: 22` → `fontSize: DesignTokens.typography.sizes.xl` (20, 差2px)
   - `fontSize: 26` → `fontSize: DesignTokens.typography.sizes['2xl']` (24, 差2px)
   - `fontSize: 28` → `fontSize: DesignTokens.typography.sizes['2xl']` (24, 差4px) 或 `DesignTokens.typography.sizes['3xl']` (30, 差2px)
   - `fontSize: 32` → `fontSize: DesignTokens.typography.sizes['3xl']` (30, 差2px)
   - `fontSize: 40` → `fontSize: DesignTokens.typography.sizes['4xl']` (36, 差4px) 或 `DesignTokens.typography.sizes['5xl']` (48, 差8px)

3. 对于 Typography.styles 中已有的组合样式（hero, h1-h6, body, caption, label, button, price），优先使用组合样式而非单独设置 fontSize

4. 添加 DesignTokens 导入

5. 如果文件已使用 `Typography.styles.*`，保持不变

注意：不替换 lineHeight 和 fontWeight 中的数字（它们不是 fontSize）
</action>

<acceptance_criteria>
- `grep -rn "fontSize:\s*[0-9]\+" apps/mobile/src/ --include="*.tsx" --include="*.ts" | grep -v "design-system/" | grep -v "node_modules/" | grep -v "lineHeight" | wc -l` 返回 0
- `cd apps/mobile && npx tsc --noEmit` exits with code 0
</acceptance_criteria>

### Task 05-02: 硬编码间距替换

<objective>
将所有硬编码间距值（padding, margin, gap, width, height 中的数字）替换为 DesignTokens.spacing 令牌引用。
</objective>

<read_first>
- apps/mobile/src/design-system/theme/tokens/design-tokens.ts (DesignTokens.spacing)
- apps/mobile/src/design-system/theme/index.ts (Spacing, SpacingAliases)
- 所有包含硬编码间距的 .tsx 文件
</read_first>

<action>
对所有文件中的硬编码间距执行替换：

1. 使用 Spacing 别名（优先）或 DesignTokens.spacing 数字键：
   - `4` → `Spacing.xs` 或 `DesignTokens.spacing[1]`
   - `8` → `Spacing.sm` 或 `DesignTokens.spacing[2]`
   - `12` → `DesignTokens.spacing[3]`
   - `16` → `Spacing.md` 或 `DesignTokens.spacing[4]`
   - `20` → `DesignTokens.spacing[5]`
   - `24` → `Spacing.lg` 或 `DesignTokens.spacing[6]`
   - `28` → `DesignTokens.spacing[7]`
   - `32` → `Spacing.xl` 或 `DesignTokens.spacing[8]`
   - `36` → `DesignTokens.spacing[9]`
   - `40` → `DesignTokens.spacing[10]`
   - `48` → `Spacing['2xl']` 或 `DesignTokens.spacing[12]`
   - `56` → `DesignTokens.spacing[14]`
   - `64` → `Spacing['3xl']` 或 `DesignTokens.spacing[16]`
   - `80` → `Spacing['4xl']` 或 `DesignTokens.spacing[20]`
   - `96` → `Spacing['5xl']` 或 `DesignTokens.spacing[24]`

2. 近似映射（不在 DesignTokens 中的值）：
   - `2` → `DesignTokens.spacing['0.5']`
   - `6` → `DesignTokens.spacing['1.5']`
   - `10` → `DesignTokens.spacing['2.5']`
   - `14` → `DesignTokens.spacing['3.5']`
   - `18` → `DesignTokens.spacing[4]` (16, 差2) 或 `DesignTokens.spacing[5]` (20, 差2)
   - `22` → `DesignTokens.spacing[5]` (20, 差2) 或 `DesignTokens.spacing[6]` (24, 差2)

3. 特殊值处理：
   - `0` → `DesignTokens.spacing[0]` 或直接 `0`
   - `1` → `DesignTokens.spacing.px`
   - 百分比值（如 `'50%'`）不替换
   - `'100%'` 不替换

4. 优先使用 Spacing 别名（xs, sm, md, lg, xl 等）而非数字键

5. 添加 Spacing 导入

注意：
- 只替换 padding/margin/gap/width/height 中的间距值
- 不替换 flex, zIndex, borderRadius, elevation, opacity 等非间距数字
- 不替换 fontSize, lineHeight, fontWeight 中的数字
- 对于 StyleSheet.create 中的样式，使用静态令牌引用
</action>

<acceptance_criteria>
- `grep -rn "padding[HTBLR]*:\s*[0-9]\+\|margin[HTBLR]*:\s*[0-9]\+\|gap:\s*[0-9]\+" apps/mobile/src/ --include="*.tsx" | grep -v "design-system/" | grep -v "node_modules/" | wc -l` 返回 0（或仅剩无法映射的特殊值）
- `cd apps/mobile && npx tsc --noEmit` exits with code 0
</acceptance_criteria>

## Verification

1. `cd apps/mobile && npx tsc --noEmit` — 零编译错误
2. 无硬编码 fontSize（用户可见组件）
3. 无硬编码间距（用户可见组件）
4. 所有替换使用 DesignTokens.typography.sizes.* 或 Spacing.* 令牌

## Success Criteria

- 用户可见组件零硬编码 fontSize
- 用户可见组件零硬编码间距
- 所有替换使用语义化令牌引用
- TypeScript 编译零错误
