# Plan 06: 移除死配置 + 最终验证

## Objective

移除 NativeWind/Tailwind 死配置，清理 colors.ts 中的 Material Design 硬编码色，执行最终编译验证和残留审计。

## Threat Model

- 无安全威胁：删除未使用的配置文件和清理令牌定义
- 风险：删除配置文件可能影响构建流程，需验证 Metro bundler 正常启动

## Tasks

### Task 06-01: 移除 NativeWind/Tailwind 死配置

<objective>
删除三个零使用的 NativeWind/Tailwind 配置文件。
</objective>

<read_first>
- apps/mobile/tailwind.config.js
- apps/mobile/nativewind.config.js
- apps/mobile/postcss.config.js
</read_first>

<action>
1. 删除以下文件：
   - `apps/mobile/tailwind.config.js`
   - `apps/mobile/nativewind.config.js`
   - `apps/mobile/postcss.config.js`

2. 检查是否有任何文件引用这些配置：
   - `grep -rn "tailwind\|nativewind\|postcss" apps/mobile/ --include="*.ts" --include="*.tsx" --include="*.json" | grep -v node_modules | grep -v ".planning/"`
   - 如果有引用，移除对应的 import/require

3. 检查 package.json 中是否有 tailwindcss/nativewind 依赖：
   - 如果有，从 `apps/mobile/package.json` 的 dependencies/devDependencies 中移除：
     - `tailwindcss`
     - `nativewind`
     - `postcss`
     - 任何 tailwind 插件

4. 运行 `cd apps/mobile && npx tsc --noEmit` 验证编译通过
</action>

<acceptance_criteria>
- `test ! -f apps/mobile/tailwind.config.js` returns exit code 0
- `test ! -f apps/mobile/nativewind.config.js` returns exit code 0
- `test ! -f apps/mobile/postcss.config.js` returns exit code 0
- `grep -rn "from 'nativewind'\|from 'tailwindcss'\|require('nativewind')\|require('tailwindcss')" apps/mobile/src/ | wc -l` returns 0
- `cd apps/mobile && npx tsc --noEmit` exits with code 0
</acceptance_criteria>

### Task 06-02: 清理 colors.ts 中的 Material Design 硬编码色

<objective>
将 colors.ts 中残留的 Material Design 硬编码颜色值替换为 DesignTokens 引用或移除未使用的色板。
</objective>

<read_first>
- apps/mobile/src/design-system/theme/tokens/colors.ts
- apps/mobile/src/design-system/theme/tokens/design-tokens.ts
- apps/mobile/src/design-system/theme/index.ts
</read_first>

<action>
1. 在 `apps/mobile/src/design-system/theme/tokens/colors.ts` 中：
   - 扫描所有 Material Design 硬编码色值（#4CAF50, #FFC107, #F44336, #2196F3 等）
   - 如果色板（如 PrimaryColors, SecondaryColors）仅在此文件定义且未被其他文件使用，标记为 @deprecated 或移除
   - 如果色板被使用，将硬编码值替换为 DesignTokens 引用

2. 特别处理：
   - `WarmPrimaryColors` — 保留，这是品牌色系
   - `BrandColors` — 保留，这是品牌色系
   - `PrimaryColors` — 如果包含 Material Design 蓝色(#2196F3)，替换为 DesignTokens.colors.brand.slate 系
   - `SecondaryColors` — 如果包含 Material Design 紫色(#9C27B0)，替换为 DesignTokens.colors.brand.terracotta 系
   - `GradientPresets` — 保留，但检查是否有紫色渐变需替换
   - `SemanticColors` — 保留，确保值与 DesignTokens.colors.semantic 一致
   - `FashionColors` — 保留，但检查是否有冲突色

3. 确保 `index.ts` 中的 `Colors` 对象不再包含 `accent` 紫色色板中的硬编码值，改为从 DesignTokens 引用
</action>

<acceptance_criteria>
- `colors.ts` 中无 Material Design 默认色硬编码（#4CAF50, #FFC107, #F44336, #2196F3, #E91E63, #9C27B0, #673AB7）
- `cd apps/mobile && npx tsc --noEmit` exits with code 0
</acceptance_criteria>

### Task 06-03: 最终残留审计 + 编译验证

<objective>
执行最终审计，确认所有硬编码值已替换，编译通过。
</objective>

<read_first>
- apps/mobile/src/design-system/theme/tokens/design-tokens.ts
- apps/mobile/src/design-system/theme/index.ts
</read_first>

<action>
1. 运行硬编码颜色审计：
   ```bash
   grep -rn "#[0-9A-Fa-f]\{6\}" apps/mobile/src/ --include="*.tsx" --include="*.ts" | grep -v "design-system/theme/tokens/" | grep -v "node_modules/" | grep -v ".planning/"
   ```
   - 仅允许的残留：rgba() 颜色（如 overlay/玻璃效果）、token 定义文件中的颜色
   - 如果有非预期的残留，逐一替换

2. 运行硬编码 fontSize 审计：
   ```bash
   grep -rn "fontSize:\s*[0-9]\+" apps/mobile/src/ --include="*.tsx" --include="*.ts" | grep -v "design-system/" | grep -v "node_modules/" | grep -v ".planning/"
   ```
   - 仅允许的残留：token 定义文件中的值
   - 如果有非预期的残留，逐一替换

3. 运行硬编码间距审计：
   ```bash
   grep -rn "padding[HTBLR]*:\s*[0-9]\+\|margin[HTBLR]*:\s*[0-9]\+\|gap:\s*[0-9]\+" apps/mobile/src/ --include="*.tsx" | grep -v "design-system/" | grep -v "node_modules/" | grep -v ".planning/"
   ```
   - 仅允许的残留：无法映射到令牌的特殊值
   - 如果有可替换的残留，逐一替换

4. 最终编译验证：
   ```bash
   cd apps/mobile && npx tsc --noEmit
   ```
   - 必须零错误

5. 紫色残留审计：
   ```bash
   grep -rn "#a855f7\|#ec4899\|#6C5CE7\|#F3F1FF\|#8B5CF6\|#7C3AED\|#6D28D9\|#5B21B6\|#4C1D95" apps/mobile/src/ --include="*.tsx" --include="*.ts" | grep -v "design-system/theme/tokens/" | grep -v "node_modules/"
   ```
   - 必须返回 0 行
</action>

<acceptance_criteria>
- 硬编码颜色审计：用户可见组件中 0 处硬编码 hex 颜色（token 定义文件除外）
- 硬编码 fontSize 审计：用户可见组件中 0 处硬编码 fontSize（token 定义文件除外）
- 紫色残留审计：0 处紫色硬编码
- `cd apps/mobile && npx tsc --noEmit` exits with code 0
- `test ! -f apps/mobile/tailwind.config.js` — 文件已删除
</acceptance_criteria>
