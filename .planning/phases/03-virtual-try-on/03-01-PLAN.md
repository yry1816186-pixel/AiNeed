   - `DOUBAO_SEEDREAM_API_KEY`（必填）
   - `DOUBAO_SEEDREAM_API_URL` = `https://visual.volcengineapi.com/v1/aigc/generate`
   - `DOUBAO_SEEDREAM_RESULT_URL` = `https://visual.volcengineapi.com/v1/aigc/result`
   - `DOUBAO_SEEDREAM_ENABLED` = `false`（默认关闭）

3. 创建 axios 实例，timeout=25000ms，headers 包含 `Authorization: Bearer {apiKey}`
4. 使用 opossum 熔断器（复用 CloudTryOnProvider 的配置模式）:
   - timeout: 25000ms
   - errorThresholdPercentage: 50
   - resetTimeout: 60000ms
   - volumeThreshold: 3
5. `isAvailable()`: enabled=true 且熔断器未打开
6. `virtualTryOn()` 核心逻辑:
   a. 将 personImageUrl 和 garmentImageUrl 转为 base64（复用 fetchImageAsBase64 方法）
   b. 构建请求体:
   ```json
   {
     "req_key": "high_aes_general_v21_L",
     "model": "doubao-seedream-3-0-t2i-250415",
     "parameters": {
       "image": "<base64_person>",
       "ref_image": "<base64_garment>",
       "prompt": "穿着这件衣服的人物照片，保持人物面部和姿势不变，高质量真实感",
       "strength": 0.75,
       "seed": -1
     }
   }
   ```
   c. POST 到 API 端点，获取 task_id
   d. 如果响应直接包含 results（同步模式），直接处理结果
   e. 如果 status="processing"（异步模式），轮询结果端点 `GET {RESULT_URL}/{task_id}`，间隔 2s，最多 10 次
   f. 从 results[0].url 下载结果图片，上传到 MinIO（复用 StorageService.uploadImage）
   g. 返回 TryOnResponse
7. 添加 `fetchImageAsBase64` 和 `downloadImage` 私有方法（从 CloudTryOnProvider 复制）
8. 添加 `pollForResult` 私有方法处理异步轮询
</action>
