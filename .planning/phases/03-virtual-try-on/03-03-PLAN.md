## Objective
增强移动端试衣体验：实时进度展示、原图与效果图对比、多方案并排对比、分享功能、试衣历史页面。

## Tasks

### Task 1: 实时进度展示增强

<read_first>
- apps/mobile/src/components/screens/TryOnScreen.tsx
- apps/mobile/src/services/websocket.ts
</read_first>

<action>
1. 修改 `websocket.ts`:
   - TryOnEventPayload 添加 `progress?: number` 和 `stage?: string` 字段
   - 添加 `try_on_progress` 事件监听:
     ```typescript
     this.socket.on("try_on_progress", (payload: TryOnProgressPayload) => {
       const listeners = this.progressListeners.get(payload.tryOnId);
       if (listeners) listeners.forEach(l => l(payload));
     });
     ```
   - 添加 `onTryOnProgress(tryOnId, listener)` 方法
   - 添加 `TryOnProgressPayload` 类型: `{ tryOnId: string; progress: number; stage: string; timestamp: string }`

2. 修改 `TryOnScreen.tsx`:
   - 添加 WebSocket 进度监听（与轮询并行）:
     ```typescript
     wsUnsubscribeRef.current = wsService.onTryOnProgress(tryOnId, (payload) => {
       setProgress(payload.progress);
       if (payload.stage === 'generating') setPhase('processing');
     });
     ```
   - 添加趣味穿搭小贴士轮播（VTO-05）:
     - 定义 tips 数组（8-10 条色彩/搭配/风格知识）
     - processing 阶段每 5 秒切换一条 tip
     - 在进度条下方显示当前 tip
   - 30 秒超时降级提示:
     - 如果 30s 后仍在 processing，显示 "AI 正在努力生成中，请稍候..."
     - 如果 60s 后仍无结果，显示 "生成时间较长，是否使用预览模式查看？"
</action>

<acceptance_criteria>
- websocket.ts 包含 try_on_progress 事件监听
- TryOnScreen 使用 WebSocket 实时更新进度
- processing 阶段显示穿搭小贴士轮播
- 30s/60s 超时降级提示存在
- TypeScript 编译无错误
</acceptance_criteria>
