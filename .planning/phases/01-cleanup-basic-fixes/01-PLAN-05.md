# Plan 05: 统一命名规范

<objective>
统一后端控制器装饰器的引号风格（单引号 → 双引号），并记录移动端 Store 命名不一致问题供后续 Phase 处理。API 端点路径不一致问题记录但不在本 Phase 修复（涉及路由变更）。
</objective>

<threat_model>
| Threat | Severity | Mitigation |
|--------|----------|------------|
| 修改控制器装饰器可能影响路由 | Low | 仅修改引号风格，不改变路由路径字符串内容 |
| Store 重命名影响大量导入 | Medium | Store 重命名推迟到 Phase 5（移动端页面重组），避免大规模导入变更 |
| API 端点路径变更影响前端调用 | High | API 端点不一致仅记录，不在本 Phase 修复 |
</threat_model>
