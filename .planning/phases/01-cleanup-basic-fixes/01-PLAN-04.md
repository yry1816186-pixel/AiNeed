# Plan 04: ESLint 配置升级

<objective>
将 ESLint no-explicit-any 规则从 warn 升级为 error（阻止新增 any 类型），并为移动端添加 recommended-requiring-type-checking 配置。对现有 any 违规使用 eslint-disable 过渡策略。
</objective>

<threat_model>
| Threat | Severity | Mitigation |
|--------|----------|------------|
| no-explicit-any: error 导致大量现有 lint 错误 | Medium | 对现有 any 添加 eslint-disable-next-line 注释，Phase 7 会修复 |
| 移动端 @typescript-eslint@^7 不完全支持 recommended-requiring-type-checking | Medium | 先尝试添加，如果类型检查规则报错则升级到 ^8 |
| 升级 @typescript-eslint 可能与 React Native 0.76.8 不兼容 | Low | 仅在必要时升级，升级前检查兼容性；回退方案：如升级失败则仅保留 no-explicit-any: error，不添加 recommended-requiring-type-checking |
</threat_model>

<read_first>

</read_first>
<action>
修改后端 ESLint 配置：

1. 将 `@typescript-eslint/no-explicit-any` 从 `"warn"` 改为 `"error"`：

   现有 ~310 处 any 将在 Phase 7 修复，当前通过 eslint-disable-next-line 豁免。

注意：后端已有 `recommended-requiring-type-checking`，无需额外添加。
</action>

</acceptance_criteria>
</task>

<task id="1-04-03" type="modify">
<read_first>
- apps/backend/.eslintrc.json
- apps/mobile/.eslintrc.json
</read_first>
<action>
为后端和移动端所有现有 no-explicit-any 违规添加 eslint-disable 注释，使 lint 命令在 no-explicit-any: error 下仍可通过。

1. 后端（~310 处 any）：
   运行 `cd apps/backend && npx eslint "src/**/*.ts" --rule '@typescript-eslint/no-explicit-any: error' -f compact 2>&1 | grep "no-explicit-any"`
   对每个报错位置，在上一行添加：
   ```typescript
   // eslint-disable-next-line @typescript-eslint/no-explicit-any
   ```
   如果同一行有多处 any，使用块级禁用：
   ```typescript
   /* eslint-disable @typescript-eslint/no-explicit-any */
   // ... 含 any 的代码
   /* eslint-enable @typescript-eslint/no-explicit-any */
   ```

2. 移动端（~29 处 any）：
   运行 `cd apps/mobile && npx eslint "src/**/*.{ts,tsx}" --rule '@typescript-eslint/no-explicit-any: error' -f compact 2>&1 | grep "no-explicit-any"`
   同样方式添加 eslint-disable 注释。

3. 对移动端 `as never` 用法（~24 处），如果被 no-explicit-any 规则捕获，
   也添加 eslint-disable 注释（这些是导航类型绕过，Phase 7 会处理）。

注意：此步骤仅添加 eslint-disable 注释，不修改任何业务逻辑。
</action>
<acceptance_criteria>
- `cd apps/backend && npx eslint "src/**/*.ts" --rule '@typescript-eslint/no-explicit-any: error' 2>&1 | grep "no-explicit-any" | wc -l` 返回 0
- `cd apps/mobile && npx eslint "src/**/*.{ts,tsx}" --rule '@typescript-eslint/no-explicit-any: error' 2>&1 | grep "no-explicit-any" | wc -l` 返回 0
</acceptance_criteria>
</task>
