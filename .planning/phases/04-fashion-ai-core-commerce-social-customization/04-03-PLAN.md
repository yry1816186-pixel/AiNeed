# Plan 03: Commerce 域迁移 + 消除 Cart↔Coupon 和 RefundRequest↔Payment 循环依赖

## must_haves

- commerce 域目录结构创建完成
- 9 个模块正确迁移到 `src/domains/commerce/` 下
- CartModule ↔ CouponModule 的 forwardRef 循环依赖已消除
- RefundRequestModule ↔ PaymentModule 的 forwardRef 循环依赖已消除
- notification + stock-notification 合并为统一消息推送模块
- `nest build` 编译通过
- 所有商业相关 API 端点正常工作

## Task 04: 消除 CartModule ↔ CouponModule 循环依赖并迁移

<read_first>
- apps/backend/src/modules/cart/cart.module.ts
- apps/backend/src/modules/cart/cart.service.ts
- apps/backend/src/modules/coupon/coupon.module.ts
- apps/backend/src/modules/coupon/coupon.service.ts
- apps/backend/src/app.module.ts
</read_first>

<action>
1. **解耦策略**：CartModule 通过 EventEmitter2 发送事件，CouponModule 监听事件

2. 修改 `src/modules/cart/cart.module.ts`：
   - 移除 `forwardRef(() => CouponModule)` 导入
   - 移除 `CouponModule` 从 imports 数组
   - 确保 `EventEmitter2` 已注册（通常通过 @nestjs/event-emitter 全局注册）

3. 修改 `src/modules/cart/cart.service.ts`：
   - 移除 `CouponService` 的直接注入
   - 在需要验证优惠券时，发送 `cart.coupon.validate` 事件
   - 在优惠券应用后，发送 `cart.coupon.applied` 事件

4. 修改 `src/modules/coupon/coupon.module.ts`：
   - 移除 `forwardRef(() => CartModule)` 导入（如果有）
   - 添加事件监听器处理优惠券验证请求

5. 复制 `src/modules/cart/` 到 `src/domains/commerce/cart/`
6. 复制 `src/modules/coupon/` 到 `src/domains/commerce/coupon/`
7. 更新两个模块中的导入路径
8. 在 `src/app.module.ts` 中更新导入路径
9. 删除 `src/modules/cart/` 和 `src/modules/coupon/` 目录
10. 运行 `cd apps/backend && npx nest build` 验证编译通过
</action>
