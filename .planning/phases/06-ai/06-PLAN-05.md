### Task 01: 统一路由错误处理

<type>refactor</type>
<files>
  - ml/api/routes/stylist.py
  - ml/api/routes/analysis.py
  - ml/api/routes/fashion_recommend.py
  - ml/api/routes/virtual_tryon.py
</files>
<read_first>
  - ml/api/middleware/error_handler.py
  - ml/api/routes/stylist.py
  - ml/api/routes/analysis.py
</read_first>
<action>
将所有路由中的 `raise HTTPException(...)` 替换为对应的 MLError 子类：

1. 服务不可用 → `raise ModelNotLoadedError(model_name="service_name")`
2. 推理失败 → `raise InferenceError(message="...")`
3. 限流 → `raise RateLimitError(retry_after=60)`
4. 参数验证 → `raise ValidationError(message="...")`

具体替换：
- `raise HTTPException(status_code=503, detail="...不可用")` → `raise ModelNotLoadedError(model_name="...")`
- `raise HTTPException(status_code=500, detail="...失败")` → `raise InferenceError(message="...")`
- `raise HTTPException(status_code=400, detail="...")` → `raise ValidationError(message="...")`
- `raise HTTPException(status_code=429)` → `raise RateLimitError()`

保留 HTTPException 仅用于纯 HTTP 层面的错误（如 404 Not Found）。
</action>
<acceptance_criteria>
  - `grep -c "raise HTTPException" ml/api/routes/stylist.py` <= 2 (仅保留 404 等 HTTP 错误)
  - `grep -c "raise HTTPException" ml/api/routes/analysis.py` <= 3
  - `grep -c "ModelNotLoadedError\|InferenceError\|ValidationError\|RateLimitError" ml/api/routes/stylist.py` >= 2
  - `grep -c "ModelNotLoadedError\|InferenceError\|ValidationError\|RateLimitError" ml/api/routes/analysis.py` >= 2
</acceptance_criteria>

### Task 02: 统一日志格式

<type>refactor</type>
<files>
  - ml/api/routes/stylist.py
  - ml/api/routes/analysis.py
  - ml/api/routes/fashion_recommend.py
  - ml/api/routes/virtual_tryon.py
</files>
<read_first>
  - ml/api/middleware/logging.py
</read_first>
<action>
统一所有路由的日志格式：

1. 所有 logger 调用使用结构化 extra 字典：
   ```python
   logger.error(
       "Stylist chat failed",
       extra={"service": "stylist", "endpoint": "chat", "error": str(e)},
   )
   ```

2. 日志级别规范：
   - `logger.info` — 请求开始/完成
   - `logger.warning` — 可恢复的降级
   - `logger.error` — 操作失败

3. 格式模板：`"{service} {action} {result}"` + extra 字典
   - 例：`logger.info("stylist chat completed", extra={"service": "stylist", "task_id": task_id})`
</action>
<acceptance_criteria>
  - `grep -c "extra={" ml/api/routes/stylist.py` >= 2
  - `grep -c "extra={" ml/api/routes/analysis.py` >= 2
</acceptance_criteria>
