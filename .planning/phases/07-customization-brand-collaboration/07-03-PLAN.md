# Plan 07-03: Mobile Customization Editor + Brand QR Scan

## Objective

Build the mobile 2D customization editor with React Native SVG/gesture-handler, brand QR code scanner, and complete customization flow frontend. The editor uses react-native-svg for rendering and react-native-gesture-handler for drag/rotate/pinch interactions.

## Context

@07-CONTEXT.md -- decisions D-01 through D-06
@apps/mobile/src/screens/CustomizationScreen.tsx -- existing customization screen
@apps/mobile/src/services/api/customization.api.ts -- existing API client

## Threat Model

| File | Surface | Mitigation |
|------|---------|------------|
| brand-qr.api.ts | QR code data | validate scanned data, sanitize URLs |
| customization.api.ts | file upload | limit file size in upload config |

### Task 1: Create Customization Editor Zustand Store

<task type="auto">
<read_first>
- apps/mobile/src/stores/aiStylistStore.ts (for Zustand pattern)
- apps/mobile/src/services/api/customization.api.ts
</read_first>
<action>
1. Create `apps/mobile/src/stores/customizationEditorStore.ts`:
   - State: selectedTemplate (null | Template), designLayers (array), selectedLayerId (null), canvasData (JSON), isLoading, isSaving, previewUrl (null), quote (null)
   - Actions:
     - `selectTemplate(template)` -- set template, reset layers
     - `addImageLayer(imageUri, width, height)` -- add image layer with default position
     - `addTextLayer(text, fontSize, color)` -- add text layer
     - `updateLayerPosition(layerId, x, y)` -- move layer
     - `updateLayerScale(layerId, scale)` -- scale layer
     - `updateLayerRotation(layerId, rotation)` -- rotate layer
     - `removeLayer(layerId)` -- delete layer
     - `reorderLayers(layerIds)` -- change z-order
     - `setSelectedLayer(layerId)` -- select for editing
     - `saveDesign()` -- call API to save design
     - `calculateQuote(printSide)` -- get auto quote from API
     - `generatePreview()` -- request AI preview
     - `submitCustomization(quoteId)` -- create customization request from design
     - `reset()` -- clear all state

2. Type definitions for:
   - `Template`: id, type, name, imageUrl, basePrice, printableArea
   - `DesignLayer`: id, type (image/text), x, y, scale, rotation, width, height, content, fontSize, color, opacity, zIndex
</action>
<acceptance_criteria>
- Store has all actions defined
- Template selection resets layers
- Layer CRUD operations update state immutably
- saveDesign calls API and updates design ID
- calculateQuote returns pricing result
- reset clears all state
</acceptance_criteria>
</task>
