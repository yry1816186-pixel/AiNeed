<action>
创建 `apps/backend/src/modules/community/content-moderation.service.ts`：

1. `moderateContent(contentType: string, contentId: string, content: string, images?: string[])`:
   - 第一步: 关键词过滤 — 调用 `aiSafetyService.quickCheck(content)`
   - 命中关键词: 设置 moderationStatus="pending", 创建 ContentModerationLog(action="auto_block")
   - 未命中: 设置 moderationStatus="approved", 创建 ContentModerationLog(action="auto_pass")
   - 未命中时: 随机抽样 (默认 10% 概率) 加入人工审核队列
   - 抽中: 创建 BullMQ 任务 `content-moderation-queue`

2. `processModerationQueue(job)`:
   - BullMQ 任务处理器
   - 调用 `aiSafetyService.validateResponse(content)` 进行 AI 深度审核
   - AI 判定违规: 设置 moderationStatus="rejected", 创建 ContentModerationLog(action="auto_block", reason=AI判定结果)
   - AI 判定安全: 保持 moderationStatus="approved"

3. `manualReview(moderatorId: string, contentId: string, contentType: string, action: string, note?: string)`:
   - action: "approve" | "reject"
   - 更新 moderationStatus
   - 创建 ContentModerationLog(action="manual_approve"|"manual_reject", moderatorId, note)
   - 如果 approve: 发送通知 content_approved
   - 如果 reject: 发送通知 content_rejected

4. `getModerationQueue(query: PaginationDto)`:
   - 查询 moderationStatus="pending" 的内容
   - 按 createdAt asc 排序（先进先审）

5. `handleReportThreshold(contentType: string, contentId: string)`:
   - 检查 ContentReport count where status != "rejected"
   - 如果 count >= 3: 设置 moderationStatus="pending", isHidden=true
   - 触发 AI 初审: 调用 processModerationQueue
   - 创建 ContentModerationLog(action="auto_block", reason="report_threshold_reached")

6. 关键词词库: 在 service 中维护 `BANNED_KEYWORDS` 数组（MVP 用硬编码基础词库，后续可迁移到数据库）

7. **[T-06-03 审核日志不可篡改]**:
   - ContentModerationLog 模型不含 updatedAt 字段（仅 createdAt）
   - ContentModerationService 中禁止对 ContentModerationLog 的 update 和 delete 操作
   - 如需修正审核结果，只能新增一条新记录（append-only 模式）
   - 在 service 中添加私有方法 `assertNoModerationLogMutation()` 作为安全断言
</action>
