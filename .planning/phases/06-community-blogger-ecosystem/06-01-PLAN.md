在 community.service.ts 中新增方法：

1. `bookmarkPost(userId: string, postId: string, bookmarked: boolean, collectionId?: string)`:
   - bookmarked=true: upsert PostBookmark (userId+postId), increment CommunityPost.bookmarkCount
   - bookmarked=false: delete PostBookmark, decrement bookmarkCount
   - 如果提供 collectionId，同时添加 WardrobeCollectionItem (itemType=post)
   - 返回 { bookmarked, bookmarkCount }

2. `sharePost(userId: string, postId: string)`:
   - increment CommunityPost.shareCount
   - 返回 { shareCount }

3. `reportContent(userId: string, dto: CreateReportDto)`:
   - 防滥用: 检查 ContentReport 是否已存在 (reporterId+contentType+contentId unique)，已存在则抛出 ConflictException
   - 防滥用: 限制每用户每日举报次数上限 20 次（Redis key `report:limit:{userId}`, TTL 86400s）
   - 创建 ContentReport 记录
   - increment CommunityPost.reportCount (如果 contentType=post)
   - 如果 reportCount >= 3: 设置 moderationStatus="pending", isHidden=true
   - 返回 { reportId, status }

4. `getTrending(query: TrendingQueryDto)`:
   - 查询 CommunityPost where hotScore > 0, orderBy hotScore desc
   - 如果 type="tags": 聚合 tags 字段，返回 { tag, count, trendDirection }
   - 返回分页结果

5. `recalculateHotScores()`:
   - 计算 hotScore = (likeCount*3 + commentCount*2 + bookmarkCount*5 + shareCount*4) * time_decay
   - time_decay = 1 / (1 + hours_since_creation / 168)
   - 批量更新所有帖子 hotScore
   - 缓存热门帖子列表到 Redis (key: `community:trending`, TTL: 300s)

在 community.controller.ts 中新增端点：

1. `POST /community/posts/:id/bookmark` — AuthGuard — body: BookmarkPostDto
2. `POST /community/posts/:id/share` — AuthGuard
3. `POST /community/reports` — AuthGuard — body: CreateReportDto
4. `GET /community/trending` — OptionalAuthGuard — query: TrendingQueryDto

在 community.module.ts 中新增导入：RedisModule
