创建 `blogger-score.service.ts`，实现博主评分与等级管理（D-01~D-06）：

1. `calculateBloggerScore(userId: string)`: 计算综合评分
   - 粉丝数维度(40%): `followerScore = min(followerCount / 5000 * 100, 100)`
   - 互动率维度(30%): `engagementRate = (likeCount + commentCount*2 + bookmarkCount*3) / max(viewCount, 1)`，归一化到 0-100
   - 内容数量维度(20%): `contentScore = min(postCount / 100 * 100, 100)`
   - 收藏率维度(10%): `bookmarkRate = totalBookmarks / max(totalViews, 1)`，归一化到 0-100
   - `compositeScore = followerScore*0.4 + engagementScore*0.3 + contentScore*0.2 + bookmarkScore*0.1`

2. `determineBloggerLevel(score: number, followerCount: number)`: 等级判定
   - score >= 80 && followerCount >= 5000 → "big_v"
   - score >= 60 && followerCount >= 500 → "blogger"
   - 其他 → null

3. `updateBloggerLevel(userId: string)`: 更新博主等级
   - 计算评分 → 判定等级 → 更新 User.bloggerLevel + User.bloggerScore
   - 如果降级：设置 7 天缓冲期（存 Redis key `blogger:grace:{userId}`，TTL 7天）
   - 缓冲期内不降级，缓冲期后仍未达标则降级

4. `@Cron('0 0 3 1 * *')` 每月1号凌晨3点执行全量重算：
   - 查询所有 bloggerLevel 不为 null 的用户
   - 逐个调用 updateBloggerLevel
   - 记录日志

5. `checkGracePeriod(userId: string)`: 检查缓冲期
   - 查 Redis `blogger:grace:{userId}` 是否存在
   - 存在则在缓冲期内，不降级

6. **[T-06-01 防篡改]** bloggerLevel/bloggerScore 字段保护:
   - 在 BloggerModule 初始化时注册 Prisma middleware 拦截 User 表的 update 操作
   - 如果 update.data 包含 bloggerLevel 或 bloggerScore 字段，且调用来源不是 BloggerScoreService（通过 context 标记判断），则移除这些字段
   - 确保 bloggerLevel/bloggerScore 仅由 cron 任务和 BloggerScoreService 方法修改

7. **[D-05 大V专属权益]** 实现大V分层权益:
   - 首页推荐位: getRecommendedPosts 方法中，大V的帖子权重额外 +20%
   - 优先审核: ContentModerationService 中，大V的内容优先进入审核队列（队列优先级标记）
   - 专属客服: 在 BloggerProduct 详情页显示客服入口（大V专属，链接到客服聊天）
</action>
