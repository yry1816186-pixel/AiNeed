<action>
Create two new services for dual-channel authentication.

**File 1: `apps/backend/src/modules/auth/services/sms.service.ts`**

NestJS injectable service with:
- Constructor injection of RedisService
- `sendVerificationCode(phone: string): Promise<void>`:
  - Generate 6-digit random code using `crypto.randomInt(100000, 999999)`
  - Store in Redis with key `sms:code:{phone}`, TTL 300 seconds (5 minutes)
  - Rate limit check: key `sms:rate:{phone}`, 1 request per 60 seconds
  - If rate limited, throw HttpException with 429 status
  - Log the code in dev mode (do NOT log in production)
  - Call SMS provider API (use console.log placeholder for now, real provider integration later)
- `verifyCode(phone: string, code: string): Promise<boolean>`:
  - Retrieve stored code from Redis key `sms:code:{phone}`
  - Use `crypto.timingSafeEqual` for comparison (prevent timing attacks)
  - Delete code after successful verification (single-use)
  - Return false if code expired, not found, or mismatched
- Follow existing NestJS @Injectable() pattern from auth module

**File 2: `apps/backend/src/modules/auth/strategies/wechat.strategy.ts`**

NestJS injectable service with:
- Constructor with config injection (WECHAT_APP_ID, WECHAT_APP_SECRET from env)
- `exchangeCodeForToken(code: string): Promise<{ accessToken: string; openid: string; unionid?: string }>`:
  - POST to `https://api.weixin.qq.com/sns/oauth2/access_token` with appid, secret, code, grant_type=authorization_code
  - Parse response for access_token, openid, unionid
  - Throw on error response (errcode != 0)
- `getUserInfo(accessToken: string, openid: string): Promise<WechatUserInfo>`:
  - GET `https://api.weixin.qq.com/sns/userinfo` with access_token and openid
  - Return nickname, headimgurl, sex, language, city, province, country

Write unit tests in:
- `apps/backend/src/modules/auth/services/sms.service.spec.ts`
- `apps/backend/src/modules/auth/strategies/wechat.strategy.spec.ts`

Tests must mock RedisService and HTTP calls.
</action>

<action>
Create three new services.

**File 1: `apps/backend/src/modules/photos/services/photo-quality-validator.service.ts`**

NestJS injectable service using `sharp` library (already in project):
- `validateQuality(imageBuffer: Buffer): Promise<PhotoQualityReport>`:
  - Clarity score: compute Laplacian variance via sharp grayscale + raw pixel data. Score 0-100.
  - Brightness score: compute mean pixel value. Score 0-100 (optimal 40-70).
  - Composition score: placeholder returning 50 (full implementation requires face detection integration with existing FaceShapeAnalyzer).
  - Return `{ clarity: number, brightness: number, composition: number, overall: number, passed: boolean }` where overall = (clarity * 0.4 + brightness * 0.3 + composition * 0.3) and passed = overall >= 40
  - All scores bounded [0, 100]

**File 2: `apps/backend/src/modules/profile/services/profile-completeness.service.ts`**

NestJS injectable service:
- `calculateCompleteness(userProfile: UserProfile): CompletenessResult`:
  - Basic info (30%): gender filled (10%), age/birthdate (10%), nickname (10%)
  - Body data (25%): height (8%), weight (8%), bodyType (9%)
  - Style preferences (20%): at least one StyleProfile entry exists (10%), at least one StylePreference (10%)
  - Color preferences (15%): colorSeason filled (8%), at least one ColorPreference (7%)
  - Photos (10%): at least one UserPhoto exists
  - Return `{ percentage: number, missingFields: string[] }` where missingFields contains human-readable Chinese labels like "身材数据", "风格偏好", "色彩分析", "个人照片"
  - percentage is integer 0-100

**File 3: `apps/backend/src/modules/style-quiz/services/quiz-progress.service.ts`**

NestJS injectable service using RedisService:
- `saveProgress(userId: string, quizId: string, questionIndex: number, answers: Record<string, string>): Promise<void>`:
  - Store in Redis key `quiz:progress:{userId}:{quizId}` as JSON, TTL 24 hours
  - Value: `{ questionIndex, answers, updatedAt: timestamp }`
- `getProgress(userId: string, quizId: string): Promise<QuizProgress | null>`:
  - Retrieve from Redis, return null if not found or expired
- `clearProgress(userId: string, quizId: string): Promise<void>`:
  - Delete the Redis key

Write unit tests for all three services.
</action>

**File 1: `apps/backend/src/modules/style-quiz/services/color-derivation.service.ts`**

NestJS injectable service:
- `deriveColorPreferences(selectedOptions: Array<{ colorTags: Array<{ hex: string; category: string; weight: number }> }>): ColorPreferenceResult`:
  - Aggregate color tags from all selected quiz options
  - Sum weights per hex code, keep highest-weight category
  - Sort by total weight descending, take top 8
  - Map hex codes to Chinese color names (provide at least 20 common mappings: #2D2D2D->炭灰, #F5F5DC->米色, #8B4513->棕色, etc.)
  - Return `{ colorPalette: Array<{ hex: string; name: string; category: string; weight: number }> }`
  - Result must be stable for same input (deterministic)

**File 2: `apps/backend/src/modules/profile/services/share-poster.service.ts`**

NestJS injectable service:
- `generatePoster(userId: string, profileData: UserProfileSummary): Promise<string>`:
  - Read ShareTemplate from database (use first active template)
  - Use node-canvas (already in project as PosterGeneratorService dependency) to render poster image with:
    - Style type name and match percentage
    - Color palette circles
    - User avatar and nickname
    - QR code placeholder
  - Upload generated PNG to MinIO (use existing upload infrastructure from Photos module)
  - Return the MinIO URL
  - node-canvas is the only poster rendering engine -- do NOT introduce Puppeteer (200MB+ anti-pattern per RESEARCH.md)

**File 3: `apps/backend/src/modules/profile/services/profile-event-emitter.service.ts`**

NestJS injectable service:
- `emitProfileUpdated(userId: string, changedFields: string[]): Promise<void>`:
  - Publish to Redis Pub/Sub channel `profile:updated`

  - Use RedisService or direct Redis client publish
- `emitQuizResultSaved(userId: string, quizId: string): Promise<void>`:
  - Publish to Redis Pub/Sub channel `quiz:completed`
