```typescript
export class PhoneLoginDto {
  @IsString()
  @IsNotEmpty()
  @Matches(/^1[3-9]\d{9}$/, { message: 'Invalid Chinese phone number' })
  phone: string;

  @IsString()
  @IsNotEmpty()
  @Length(6, 6)
  code: string;
}

export class PhoneRegisterDto {
  @IsString()
  @IsNotEmpty()
  @Matches(/^1[3-9]\d{9}$/, { message: 'Invalid Chinese phone number' })
  phone: string;

  @IsString()
  @IsNotEmpty()
  @Length(6, 6)
  code: string;

  @IsEnum(Gender)
  @IsNotEmpty({ message: 'Gender is required for personalized recommendations' })
  gender: Gender;

  @IsOptional()
  @IsString()
  birthDate?: string; // Used to derive age range

  @IsString()
  @IsOptional()
  @Length(2, 20)
  nickname?: string;
}

export class WechatLoginDto {
  @IsString()
  @IsNotEmpty()
  code: string;
}
```

Follow the existing DTO patterns in the file (same imports, decorator style, ApiProperty if used).
</action>

<action>
Create a seed script that populates the database with a default style quiz question bank.

**File: `apps/backend/prisma/seed-style-quiz.ts`**

The script must:
1. Import PrismaClient from `@prisma/client`
2. Clear existing quiz data idempotently (delete QuizAnswer, StyleQuizResult, StyleQuiz in order, then QuizOption, QuizQuestion)
3. Create 1 quiz with 6 questions, each with 4-6 image options
4. Each option must have:
   - `imageUrl`: placeholder URL (e.g., `https://placeholder.com/quiz/q{N}a{M}.jpg`)
   - `label`: Chinese label for the style shown
   - `styleScores`: JSON with style-to-weight mapping (e.g., `{ "MINIMALIST": 3, "CASUAL": 1 }`)
   - `colorTags`: JSON array with hex/category/weight (e.g., `[{ "hex": "#2D2D2D", "category": "NEUTRAL", "weight": 2 }]`)
   - `order`: display order within question
5. Questions cover: daily outfit preference, color combination, pattern style, silhouette preference, accessory style, lifestyle scene
6. Include error handling with `process.exit(1)` on failure
7. Call `prisma.$disconnect()` in finally block

Reference the RESEARCH.md "Pattern 2: Style Quiz Scoring Algorithm" section for the styleScores/colorTags JSON structure.
</action>
