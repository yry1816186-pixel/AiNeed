### Task 1: Extend AuthService and AuthController for Phone/WeChat Login

<type>extend</type>

</read_first>
<action>
Read the existing auth.service.ts and auth.controller.ts to understand current patterns.

**Extend AuthService** (`auth.service.ts`):
- Inject SmsService and WechatAuthStrategy via constructor
- Add `loginWithPhone(phone: string, code: string): Promise<AuthResponse>`:
  - Call smsService.verifyCode(phone, code)
  - If verified, find user by phone field in database
  - If user not found, throw UnauthorizedException
  - Generate and return JWT access + refresh tokens
- Add `registerWithPhone(phone: string, code: string, nickname?: string): Promise<AuthResponse>`:
  - Call smsService.verifyCode(phone, code)
  - If verified, check if user with phone already exists
  - If exists, throw ConflictException
  - Create new User with authProvider='phone', phone=phone
  - Create UserProfile with nickname if provided
  - Generate and return JWT tokens
- Add `loginWithWechat(code: string): Promise<AuthResponse>`:
  - Call wechatAuthStrategy.exchangeCodeForToken(code)
  - Find user by wechatOpenId
  - If user not found, create new User with authProvider='wechat', wechatOpenId, wechatUnionId
  - Create UserProfile from WeChat user info (nickname, avatar)
  - Generate and return JWT tokens
- Add mandatory gender field to register flow:
  - RegisterDto (already has gender from Plan 01 Task 2's PhoneRegisterDto extension)
  - In registerWithPhone, gender is MANDATORY (enforced by PhoneRegisterDto @IsNotEmpty()). Always set on UserProfile. Throw if missing.

**Extend AuthController** (`auth.controller.ts`):
- Add `POST /auth/phone-login` endpoint:
  - Accepts PhoneLoginDto
  - Calls authService.loginWithPhone
  - Rate limited: @Throttle({ default: { limit: 5, ttl: 60000 } }) (5 requests per minute)
  - @Public() decorator (no auth required)
- Add `POST /auth/phone-register` endpoint:
  - Accepts PhoneRegisterDto
  - Calls authService.registerWithPhone
  - Rate limited: same 5/min
  - @Public() decorator
- Add `POST /auth/wechat` endpoint:
  - Accepts WechatLoginDto
  - Calls authService.loginWithWechat
  - Rate limited: same 5/min
  - @Public() decorator
- Add `POST /auth/send-code` endpoint:
  - Accepts { phone: string }
  - Calls smsService.sendVerificationCode
  - Rate limited: 1 per 60 seconds
  - @Public() decorator
- All new endpoints use `/api/v1/auth/` prefix per CC-08
- All responses follow JSON:API format per CC-09

Write integration test covering phone register -> login flow and WeChat login flow (mock WeChat API).
</action>

  - On receiving event, invalidate cached recommendations for that user
  - Subscribe to Redis Pub/Sub channel `quiz:completed`
  - On receiving event, trigger recommendation recalculation

Keep subscriptions lightweight -- do not block the event publisher.
