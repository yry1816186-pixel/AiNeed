# Phase 9 Plan 04: Performance Optimization (Backend Cache + Mobile Perf)

## Objective
Optimize backend API response caching with Redis for hot data (product details, recommendation lists), add performance monitoring interceptor, and implement mobile-side optimizations: virtualized lists, image optimization, and bundle splitting.

## Context
- CacheService already exists with getOrSet/getWithLock methods
- CacheKeyBuilder and CACHE_TTL constants exist in cache.constants.ts
- ClothingService already uses cache for some queries
- MetricsService tracks HTTP request duration with prom-client histograms
- Mobile uses React Native with Metro bundler
- No FlashList or virtualization currently implemented

## Tasks

<action>
Enhance cache constants with performance-specific TTLs and key patterns:

   - product.updated -> 'cache:product:*'
   - order.created -> 'cache:dashboard:*'
   - user.profile_updated -> 'cache:profile:{userId}*'
</action>

### Task 2: Add Cache Interceptor

<task type="auto">
<read_first>
  - apps/backend/src/common/interceptors/ (list files)
</read_first>
<action>
Create `apps/backend/src/common/interceptors/cache.interceptor.ts`:

1. NestJS Interceptor that:
   - Reads cache key from custom decorator `@CacheKey('pattern')` and TTL from `@CacheTTL(300)`
   - On GET requests: checks Redis cache, returns cached if hit, otherwise calls handler and caches result
   - On mutation (POST/PUT/PATCH/DELETE): invalidates related cache patterns
   - Uses CacheService.getOrSet for cache penetration protection
   - Sets X-Cache header: HIT/MISS for debugging

2. Create decorators in `apps/backend/src/common/decorators/cache.decorators.ts`:
   - @CacheKey(key: string) -- set cache key pattern
   - @CacheTTL(ttl: number) -- set TTL in seconds
   - @CacheClear(patterns: string[]) -- invalidate patterns after execution
</action>
<acceptance_criteria>
  - CacheInterceptor with proper cache-aside pattern
  - Decorators for declarative cache configuration
  - X-Cache header for debugging
</acceptance_criteria>
</task>
