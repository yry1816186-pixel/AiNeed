# Phase 9 Plan 01: Admin Module Foundation (RBAC + Audit + Config + Dashboard)

## Objective
Build the admin module foundation: RBAC role system, operation audit log, admin dashboard stats, and config management endpoints. This provides the base for all other admin operations.

## Context
- AdminGuard already exists at `apps/backend/src/common/guards/admin.guard.ts` but only checks `role === 'admin'`
- AdminCommunityController already exists with content moderation endpoints
- MetricsService has prom-client metrics for business events
- FeatureFlagService provides flag CRUD + evaluation
- Need to extend the admin module with RBAC (superadmin/ops/customer-service/reviewer roles), audit logging, dashboard stats, and system config management

## Tasks

### Task 1: Extend Prisma Schema for Admin RBAC + Audit Log

<task type="auto">
<read_first>
  - apps/backend/prisma/schema.prisma
</read_first>
<action>
Add to Prisma schema:

1. Update User model `role` field comment to document all admin roles: `admin`, `superadmin`, `ops`, `customer_service`, `reviewer`

2. Create `AdminAuditLog` model:
   - id (UUID, @default(uuid()))
   - userId (String, map to User)
   - action (String) -- e.g. "user.ban", "post.delete", "config.update"
   - resource (String) -- e.g. "user", "post", "feature_flag"
   - resourceId (String, optional)
   - details (Json, optional) -- before/after snapshot
   - ipAddress (String, optional)
   - userAgent (String, optional)
   - createdAt (DateTime, @default(now()))

3. Create `SystemConfig` model:
   - id (UUID, @default(uuid()))
   - key (String, @unique)
   - value (Json)
   - description (String, optional)
   - updatedBy (String, optional)
   - updatedAt (DateTime, @updatedAt)
   - createdAt (DateTime, @default(now()))
</action>
<acceptance_criteria>
  - AdminAuditLog model created with all fields
  - SystemConfig model created with unique key constraint
  - Schema compiles without errors
</acceptance_criteria>
</task>
