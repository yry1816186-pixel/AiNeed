Enhance `apps/backend/src/modules/merchant/merchant.service.ts`:

1. **Auto-validation in `applyForMerchant`** per D-09:
   - Validate business license format: Chinese unified social credit code (18 chars, regex `^[0-9A-HJ-NP-RTUW-Y]{2}\d{6}[0-9A-HJ-NP-RTUW-Y]{10}$`)
   - Validate phone number: Chinese mobile (11 digits, starts with 1, regex `^1[3-9]\d{9}$`)
   - Validate brand name uniqueness: check Brand table for existing name
   - If all pass, create Brand with verified=false (pending review)
   - If any fail, throw BadRequestException with specific validation error message

2. **Merchant review system**:
   - `getPendingApplications()`: List Brands where verified=false and isActive=true (admin endpoint)
   - `approveMerchant(brandId)`: Set Brand.verified=true, emit MERCHANT_APPROVED event
   - `rejectMerchant(brandId, reason)`: Set Brand.isActive=false, store rejection reason in Brand.description or a new field, emit MERCHANT_REJECTED event
   - `getMerchantApplication(brandId)`: Get single application details for review

3. **Merchant order management**:
   - `getMerchantOrders(brandId, options: { status?, page?, limit? })`: Find orders containing items from this brand (join OrderItem -> ClothingItem -> Brand). Support status filter and pagination.
   - `shipOrder(brandId, orderId, trackingNumber, carrier)`: Verify order contains this brand's items, verify order status is PAID or PROCESSING, update order status to SHIPPED, store tracking info. Emit ORDER_SHIPPED event.

4. **Stock management**:
   - `updateStock(brandId, itemId, stock)`: Verify item belongs to this brand, update ClothingItem.stock. After update, call StockNotificationService.checkAndNotify(itemId) and checkLowStock(itemId) (inject StockNotificationService).
   - `getLowStockItems(brandId)`: List items where brand matches and stock <= lowStockThreshold.

Add DTOs to merchant/dto/index.ts:
- ShipOrderDto: trackingNumber (string), carrier (string)
- UpdateStockDto: stock (number, @Min(0))

Add controller endpoints to merchant.controller.ts:
- GET /api/v1/merchants/applications (admin) -- list pending

- PATCH /api/v1/merchants/applications/:id/approve (admin)

- GET /api/v1/merchants/:id/orders (merchant) -- list orders
- POST /api/v1/merchants/:id/orders/:orderId/ship (merchant) -- ship order
- PATCH /api/v1/merchants/:id/items/:itemId/stock (merchant) -- update stock
