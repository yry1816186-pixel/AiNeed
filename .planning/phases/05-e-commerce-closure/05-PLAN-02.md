# Plan 05-02: Backend Enhancement — Existing Module Upgrades

<objective>
Enhance existing backend modules to support Phase 5 requirements: cart coupon integration, order refund flow, merchant review system, search enhancements, and clothing service improvements.
</objective>

<threat_model>
| Threat | Severity | Mitigation |
|--------|----------|------------|
| Cart price manipulation | High | Server-side price calculation, never trust client prices |
| Merchant privilege escalation | High | Verify merchant.verified before allowing product/order operations |
| Search injection | Medium | Parameterized Prisma queries, input sanitization |
</threat_model>

<action>
Enhance `apps/backend/src/modules/cart/cart.service.ts`:

1. **Add stock validation to `addItem`**: Before adding item, check `ClothingItem.stock >= quantity`. If stock insufficient, throw BadRequestException("库存不足")

2. **Add coupon to `getCartSummary`**: Add optional `couponCode` parameter. If provided:
   - Call `couponService.validateCoupon(code, userId, totalAmount)`
   - Call `couponService.calculateDiscount(coupon, totalAmount)`
   - Return summary with: `{ totalItems, selectedItems, totalAmount, selectedAmount, discountAmount, shippingFee, finalAmount }`
   - Shipping fee logic: finalAmount >= 99 ? 0 : 10

3. **Add `getInvalidItems(userId)`**: Find cart items where item.isActive=false OR item.isDeleted=true, return list for frontend to display/handle

4. **Add `checkPriceChanges(userId)`**: Compare cart item prices with current ClothingItem prices, return list of changed items with old/new prices

5. **Add `batchDelete(userId, cartItemIds: string[])`**: Delete multiple cart items at once (for D-14 edit mode)

6. **Add `moveToFavorites(userId, cartItemIds: string[])`**: Move selected cart items to favorites, then remove from cart
</action>
<acceptance_criteria>
- `cart.service.ts` `addItem` method checks stock before adding
- `getCartSummary` returns discountAmount and finalAmount fields
- `getInvalidItems` method exists and returns inactive/deleted items
- `batchDelete` method exists and deletes multiple items
- `moveToFavorites` method exists and moves items to favorites
</acceptance_criteria>
</task>

<action>
Enhance `apps/backend/src/modules/merchant/merchant.service.ts`:

1. **Add merchant review system**:
   - `getPendingApplications()`: List merchants with verified=false (admin endpoint)
   - `approveMerchant(brandId: string)`: Set Brand.verified=true, emit MERCHANT_APPROVED event
   - `rejectMerchant(brandId: string, reason: string)`: Set Brand.verified=false, store rejection reason, emit MERCHANT_REJECTED event
   - `getMerchantApplication(brandId: string)`: Get application details for review

2. **Add auto-validation in `applyForMerchant`** per D-09:
   - Validate business license format (Chinese unified social credit code: 18 chars, regex: `^[0-9A-HJ-NP-RTUW-Y]{2}\d{6}[0-9A-HJ-NP-RTUW-Y]{10}$`)
   - Validate phone number format (Chinese mobile: 11 digits starting with 1)
   - Validate brand name uniqueness
   - If all pass, create with verified=false (pending review)
   - If any fail, throw BadRequestException with specific validation error

3. **Add merchant order management**:
   - `getMerchantOrders(brandId: string, options)`: List orders containing this brand's items
   - `shipOrder(brandId: string, orderId: string, trackingNumber: string, carrier: string)`: Set order status to SHIPPED, add tracking info

4. **Add stock management**:
   - `updateStock(brandId: string, itemId: string, stock: number)`: Update item stock, trigger stock notification check
   - `getLowStockItems(brandId: string)`: List items where stock <= lowStockThreshold
</action>
<acceptance_criteria>
- `merchant.service.ts` contains `getPendingApplications` method
- `merchant.service.ts` contains `approveMerchant` method that sets verified=true
- `merchant.service.ts` contains `rejectMerchant` method with reason parameter
- `applyForMerchant` validates business license format, phone format, brand name uniqueness
- `getMerchantOrders` method exists
- `shipOrder` method exists and sets order status to SHIPPED
- `updateStock` method exists and triggers notification check
</acceptance_criteria>
