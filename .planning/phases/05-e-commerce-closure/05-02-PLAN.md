Step 4 -- Create `apps/backend/src/modules/refund-request/refund-request.service.ts`:
Inject PrismaService, PaymentService (via forwardRef).

Methods:
1. `create(userId, dto: CreateRefundRequestDto)` -- Validate: order belongs to userId, order status is PAID or SHIPPED (per D-17: refund within 7 days of completion for completed orders, but PAID/SHIPPED also allowed). Amount auto-calculated from order items. Create RefundRequest with status=PENDING. Use transaction.
2. `approve(refundRequestId, adminNote?)` -- Validate status=PENDING. For RETURN_REFUND type: user must have already provided trackingNumber (status must already be PROCESSING or user added tracking). Set status=APPROVED.
3. `reject(refundRequestId, adminNote)` -- Validate status=PENDING. Set status=REJECTED, store adminNote.
4. `addTrackingNumber(refundRequestId, userId, trackingNumber)` -- Validate: belongs to user, type=RETURN_REFUND, status=APPROVED. Set trackingNumber, status=PROCESSING.
5. `completeRefund(refundRequestId)` -- Validate status=APPROVED (REFUND_ONLY) or PROCESSING (RETURN_REFUND). Call PaymentService.refund with original payment record. Set status=COMPLETED, processedAt=now. Emit REFUND_COMPLETED event.
6. `getUserRefundRequests(userId, orderId?)` -- List user's refund requests with order details.
7. `getOrderRefundRequests(orderId)` -- List all refund requests for an order (merchant/admin).

Status machine per D-17:
- REFUND_ONLY: PENDING -> APPROVED -> COMPLETED (or REJECTED from PENDING)
- RETURN_REFUND: PENDING -> APPROVED -> PROCESSING -> COMPLETED (or REJECTED from PENDING/APPROVED)

Step 5 -- Create `apps/backend/src/modules/refund-request/refund-request.controller.ts`:

- PATCH /api/v1/refund-requests/:id/approve -- Approve (admin/merchant)
- PATCH /api/v1/refund-requests/:id/reject -- Reject (admin/merchant)
- PATCH /api/v1/refund-requests/:id/complete -- Complete refund (admin/merchant)
