# Plan 05-01: Backend New Modules — Coupon, StockNotification, RefundRequest, SizeRecommendation

<objective>
Create 4 new backend modules for Phase 5: Coupon/Promotion system, Stock Notification, Refund Request, and AI Size Recommendation. Includes Prisma schema additions and full CRUD services.
</objective>

<threat_model>
| Threat | Severity | Mitigation |
|--------|----------|------------|
| Coupon brute-force enumeration | Medium | Rate limit coupon validation, use 8+ char random codes |
| Refund amount manipulation | High | Server-side amount validation ≤ paid amount, status machine enforcement |
| Stock notification spam | Low | Per-user per-item subscription limit (1 per SKU) |
| Size recommendation data leak | Medium | Only return recommendation for own user, no raw body data exposed |
</threat_model>

   - POST /api/v1/refund-requests — Create refund request

   - PATCH /api/v1/refund-requests/:id/complete (admin/merchant) — Complete refund
