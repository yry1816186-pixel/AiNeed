# Plan 05-04: Frontend New Screens & Shared Components

<objective>
Create new screens and shared components for Phase 5: merchant application, stock notification management, and reusable UI components (SKU selector, coupon selector, refund form, order timeline, payment waiting).
</objective>

<threat_model>
| Threat | Severity | Mitigation |
|--------|----------|------------|
| Merchant application data leak | Medium | PII encrypted at rest, only admin can view full details |
| Stock notification spam | Low | Client-side rate limiting on subscribe button |
</threat_model>

<tasks>

<task id="05-04-01" type="feature">
<read_first>
- apps/mobile/src/screens/ (existing screen patterns)
- apps/mobile/src/services/api/commerce.api.ts
- .planning/phases/05-e-commerce-closure/05-UI-SPEC.md (Merchant Application contract)
- .planning/phases/05-e-commerce-closure/05-CONTEXT.md (D-09, D-11)
</read_first>
<action>
Create `apps/mobile/src/screens/MerchantApplyScreen.tsx`:

1. **Application Form** with fields:
   - 品牌名称: TextInput with uniqueness validation (debounced API call)
   - 营业执照号: TextInput with format validation (18 chars, regex display)
   - 联系人: TextInput
   - 手机号: TextInput with Chinese mobile format validation
   - 品牌简介: TextInput multiline (max 500 chars)
   - 品牌Logo: Image picker (optional)

2. **Auto-validation per D-09**: Real-time inline validation:
   - 营业执照号: Show ✓/✗ as user types
   - 手机号: Show ✓/✗ format check
   - 品牌名: Show "品牌名已存在" if duplicate (debounced 500ms)

3. **Submit**: "提交申请" button. On success, show status page: "申请已提交，我们将在1-3个工作日内完成审核"

4. **Status Page**: Show application status (审核中/已通过/已拒绝 + reason if rejected)

5. **API additions** — Create `apps/mobile/src/services/api/merchant.api.ts`:
   - `merchantApi.apply(data: MerchantApplyDto)` — POST /api/v1/merchants/apply
   - `merchantApi.getApplicationStatus()` — GET /api/v1/merchants/application
</action>
<acceptance_criteria>
- MerchantApplyScreen.tsx exists with form fields: 品牌名称, 营业执照号, 联系人, 手机号, 品牌简介
- Real-time validation shows ✓/✗ for 营业执照号 and 手机号
- 品牌名 has debounced uniqueness check
- Submit button shows loading state
- Status page shows 审核中/已通过/已拒绝
- merchant.api.ts exists with apply and getApplicationStatus methods
</acceptance_criteria>
</task>

<task id="05-04-03" type="feature">
<read_first>

</read_first>
<action>
Create shared components:

1. **`apps/mobile/src/components/SkuSelector.tsx`**:
   - Props: item (with colors, sizes, stock), selectedColor, selectedSize, onColorChange, onSizeChange, aiRecommendation?
   - Color swatches: Circular buttons, selected with border highlight
   - Size buttons: Square buttons, out-of-stock grayed out with "缺货" label
   - AI推荐 badge: Green badge next to recommended size, tap to expand reason
   - Quantity selector: -/N/+ buttons, max = stock for selected SKU

2. **`apps/mobile/src/components/CouponSelector.tsx`**:
   - Props: coupons[], selectedCouponId, onSelect
   - List of available coupons with: discount amount, min order, expiry date, applicable scope
   - Selected state with checkmark
   - "不使用优惠券" option at top
   - Unavailable coupons grayed out with reason ("未满足最低消费"/"已过期"/"不适用于当前商品")

3. **`apps/mobile/src/components/RefundRequestForm.tsx`**:
   - Props: orderItem, onSubmit, onCancel
   - Type selector: 仅退款 / 退货退款 (radio buttons)
   - Reason dropdown: 不想要了/商品质量问题/与描述不符/其他
   - Description: TextInput multiline
   - Image upload: Up to 3 images, tap to add from camera/album
   - Amount display: Auto-calculated, not editable
   - Submit button: "提交申请"

4. **`apps/mobile/src/components/OrderTimeline.tsx`**:
   - Props: timeline events array [{status, time, description, trackingNumber?}]
   - Vertical line with nodes
   - Current status: filled circle + accent color
   - Past status: filled circle + gray
   - Future status: outlined circle + light gray
   - Tracking number: tappable, copy to clipboard

5. **`apps/mobile/src/components/PaymentWaitingScreen.tsx`**:
   - Props: orderId, onSuccess, onTimeout
   - Loading animation (circular spinner)
   - "正在查询支付结果..." text
   - Auto-poll every 2 seconds (max 60 times)
   - Success: checkmark animation → call onSuccess
   - Timeout: "支付超时" message + "查看订单" button → call onTimeout
</action>
<acceptance_criteria>
- SkuSelector.tsx exists with color swatches, size buttons, AI推荐 badge, quantity selector
- CouponSelector.tsx exists with coupon list, selected state, unavailable state
- RefundRequestForm.tsx exists with type selector, reason dropdown, image upload
- OrderTimeline.tsx exists with vertical line, status nodes, tracking number
- PaymentWaitingScreen.tsx exists with loading, polling, success/timeout states
</acceptance_criteria>
