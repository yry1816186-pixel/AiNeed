<action>
Create a migration mapping file at `apps/mobile/src/MIGRATION_MAP.md` that documents the source → destination for every file that needs to move. This file serves as the reference for all subsequent migration tasks.

The mapping must include:
1. Every screen file in `screens/` → target `features/*/screens/`
2. Every component subdirectory in `components/` → target `features/*/components/` or `shared/components/` or `design-system/`
3. Every store file in `stores/` → target `features/*/stores/`
4. Every service file in `services/` → target `features/*/services/` or `shared/services/`
5. Every hook file in `hooks/` → target `features/*/hooks/` or `shared/hooks/`
6. Every type file in `types/` → target `features/*/types/` or `shared/types/`
7. Theme files → target `design-system/theme/`

Format each entry as:
```
SOURCE: src/screens/LoginScreen.tsx → DEST: src/features/auth/screens/LoginScreen.tsx
```

Classify each component directory as:
- **Feature-specific** → `features/*/components/`
- **Shared** → `shared/components/`
- **Design system** → `design-system/`

Use the classification from RESEARCH.md Section 1.3.
</action>
