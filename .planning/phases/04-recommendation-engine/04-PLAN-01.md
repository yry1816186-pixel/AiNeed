# Plan 01: Infrastructure Foundation — Docker + Schema + Config

<objective>
Establish the infrastructure foundation for the recommendation engine: deploy Neo4j graph database and Qdrant vector search engine via Docker, extend Prisma schema with unified behavior tracking and recommendation cache models, and configure environment variables. All subsequent algorithm and API plans depend on these services and data models existing.

Purpose: Infrastructure must be ready before any algorithm service can connect to Neo4j, Qdrant, or SASRec. The unified behavior tracking model replaces the current dual-write pattern (UserBehavior + UserBehaviorEvent) with a single event-sourced model supporting time decay. The recommendation cache model enables pre-computed results for feed pagination.

Output: Running Neo4j + Qdrant containers, extended Prisma schema, environment configuration, and module wiring.
</objective>

### Task 2: Extend Prisma Schema with Unified Behavior Tracking and Recommendation Cache

<type>extend</type>
<files>

</files>
<read_first>
- apps/backend/prisma/schema.prisma

<action>
Read the existing `apps/backend/prisma/schema.prisma` to find the UserBehavior model and understand the current schema structure.

**1. Add `InteractionWeight` enum** (for D-06 behavior weight gradient):
```prisma
enum InteractionWeight {
  view        // weight: 1
  click       // weight: 2
  like        // weight: 3
  favorite    // weight: 4
  addToCart   // weight: 5
  purchase    // weight: 8
  tryOn       // weight: 6
  share       // weight: 4
  dislike     // weight: -2
}
```

**2. Add `UserBehaviorEvent` model** (unified event-sourced behavior tracking with time decay):
```prisma
model UserBehaviorEvent {
  id          String   @id @default(cuid())
  userId      String
  itemId      String?
  itemType    String   @default("clothing")  // clothing, outfit, post
  action      InteractionWeight
  value       Float    @default(1)           // weighted value after time decay
  rawValue    Float    @default(1)           // original weight before decay
  context     Json?                           // {occasion, source, duration, etc.}
  sessionId   String?
  createdAt   DateTime @default(now())

  user        User     @relation(fields: [userId], references: [id], onDelete: Cascade)

  @@index([userId, createdAt])
  @@index([itemId, action])
  @@index([userId, action, createdAt])
  @@map("user_behavior_events")
}
```

**3. Add `RecommendationCache` model** (pre-computed recommendation results):
```prisma
model RecommendationCache {
  id          String   @id @default(cuid())
  userId      String
  category    String   // daily, occasion, trending, explore
  subCategory String?  // commute, date, sport, etc. for occasion
  results     Json     // [{itemId, score, reasons, colorHarmony}]
  version     String   @default("v1")        // cache invalidation version
  expiresAt   DateTime
  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt

  user        User     @relation(fields: [userId], references: [id], onDelete: Cascade)

  @@unique([userId, category, subCategory, version])
  @@index([userId, category, expiresAt])
  @@map("recommendation_caches")
}
```

**4. Add `KnowledgeGraphEntity` model** (sync tracking for Neo4j):
```prisma
model KnowledgeGraphEntity {
  id          String   @id @default(cuid())
  entityType  String   // Category, Style, Occasion, Brand, ColorSeason, ClothingItem
  entityId    String   // reference to source entity
  syncedAt    DateTime @default(now())
  createdAt   DateTime @default(now())

  @@unique([entityType, entityId])
  @@index([entityType])
  @@map("knowledge_graph_entities")
}
```

**5. Add relation to User model** — add these fields if not already present:
```prisma
  behaviorEvents     UserBehaviorEvent[]
  recommendationCaches RecommendationCache[]
```

After editing, run: `cd apps/backend && npx prisma validate` to confirm schema is valid.
